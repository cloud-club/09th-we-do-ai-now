## 2. 선택한 키워드 정리

### 1. Multiagent Systems
- **간단 설명**  
여러 AI 에이전트가 역할을 나누어 조사, 비교, 문서 작성, 의사결정 보조 같은 작업을 단계적으로 수행하는 흐름입니다.

- **관련 AI 용어**
  - **AI Agent**: 특정 목표를 위해 작업을 수행하는 AI 단위
  - **Orchestration**: 여러 작업이나 에이전트를 순서와 규칙에 따라 연결하는 방식
  - **Automation**: 반복 작업을 자동으로 처리하는 방식

- **Pre-Sales 직무 관점**  
고객도 AI를 활용해 기본 제품 조사, 경쟁사 비교, 질문 리스트 작성, 초안 문서 정리까지 빠르게 할 수 있게 됩니다.  
그만큼 단순 정보 전달이나 기술 설명 중심의 가치만으로는 차별화가 어려워질 수 있습니다.

### 2. Domain-Specific Language Models (DSLMs)
- **간단 설명**  
범용 모델이 아니라 특정 산업, 도메인, 업무 맥락에 맞춰 더 정확한 답변과 추론을 제공하는 특화형 AI 모델입니다.

- **관련 AI 용어**
  - **LLM**: 대규모 언어 모델
  - **Fine-tuning**: 특정 데이터로 모델을 추가 학습해 성능을 높이는 방식
  - **RAG (Retrieval-Augmented Generation)**: 외부 문서나 지식 기반을 참고해 답변 정확도를 높이는 방식

- **Pre-Sales 직무 관점**  
이제 중요한 것은 단순히 기술을 많이 아는 것이 아니라, 고객 산업과 업무 맥락에 맞게 기술을 해석하고 번역하는 능력입니다.  
Pre-Sales의 역할은 제품 설명자보다, 고객 상황에 맞는 해답을 설계하는 사람에 더 가까워질 것 같습니다.

### 3. AI Security Platforms
- **간단 설명**  
AI 도입이 확산될수록 데이터 유출, 모델 오남용, 프롬프트 공격, 거버넌스 같은 위험을 관리하기 위한 보안 및 통제 체계가 중요해지는 흐름입니다.

- **관련 AI 용어**
  - **AI Governance**: AI를 안전하고 책임 있게 운영하기 위한 관리 체계
  - **Prompt Injection**: 악의적 입력으로 AI 동작을 왜곡시키는 공격 방식
  - **Data Leakage**: 민감한 정보가 외부로 노출되는 문제

- **Pre-Sales 직무 관점**  
고객은 이제 “이 기능이 되느냐”만 묻지 않고 “안전하게 도입할 수 있느냐”를 함께 봅니다.  
따라서 Pre-Sales는 기능 설명뿐 아니라 보안, 통제, 운영 안정성까지 설명할 수 있어야 한다고 생각합니다.

## 3. 왜 이 키워드를 선택했는가

### ❓ 왜 이 키워드를 선택했는가?
이 세 가지 키워드는 공통적으로 **고객이 기술을 이해하고 구매를 판단하는 방식**을 바꾸기 때문에 선택했습니다.  
특히 Pre-Sales 관점에서는 AI 때문에 직무 자체가 갑자기 크게 바뀐다기보다는, 기존에 강점으로 여겨졌던 **기술 정보 비대칭**이나 **기술적 헤지**의 가치가 약해질 수 있다고 느꼈습니다.

예전에는 기술을 많이 알고 있다는 것만으로도 설명과 설득에서 우위가 있었지만, 이제는 고객도 AI를 통해 빠르게 기본 정보를 얻고 비교할 수 있습니다.  
그래서 앞으로는 단순한 기술 설명보다, **고객 맥락에 맞는 해석과 신뢰 형성, 도입 방향 정리**가 더 중요한 가치가 될 것 같습니다.

### ❓ 이 기술이 내 직무와 어떤 부분에서 연결되는가?
- **Multiagent Systems**는 고객의 정보 탐색과 문서 초안 작성 속도를 높여, Pre-Sales의 기본 설명 역할을 상대적으로 약하게 만들 수 있습니다.
- **DSLMs**는 도메인 이해가 더 중요한 시대를 만들기 때문에, Pre-Sales에게 고객 산업과 업무를 정확히 해석하는 역량을 더 요구합니다.
- **AI Security Platforms**는 기능 설명을 넘어, 실제 도입 리스크와 통제 방안까지 함께 설명해야 하는 역할을 강화합니다.

### ❓ 이 기술이 확산되면 내 직무의 어떤 업무가 변화할 것 같은가?
기본 제품 설명, 경쟁사 비교, 초안 제안서 작성 같은 업무는 AI의 도움으로 훨씬 빨라지고 일부는 고객이나 내부 조직에서도 직접 처리할 수 있을 것 같습니다.

반면에 Pre-Sales에게 더 중요해질 업무는 다음과 같다고 생각합니다.
- 고객의 진짜 문제를 정의하는 일
- 고객 환경에 맞는 아키텍처와 도입 방향을 설계하는 일
- 기술적 리스크와 운영 관점을 함께 설명하는 일
- 여러 이해관계자가 납득할 수 있도록 신뢰를 만드는 일

즉, 제 직무는 없어지기보다는 **기술 설명 중심 역할에서 고객 의사결정 지원 역할로 더 선명해질 것**이라고 생각합니다.

## 4. 나의 산업 / 직무 정의
