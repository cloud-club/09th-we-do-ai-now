## 2. 선택한 키워드 정리

### 1. Harness
- **간단 설명**
AI나 자동화 도구를 단순히 써보는 수준이 아니라, 실제 업무 흐름 안에 묶어서 반복 가능하게 활용하는 실행 체계를 의미한다고 이해했습니다.

- **관련 AI 용어**
  - **Workflow**: 여러 작업 단계를 연결한 실행 흐름
  - **Automation**: 반복 작업을 자동화하는 방식
  - **Operationalization**: 실험을 실제 업무에 정착시키는 과정

- **Pre-Sales 직무 관점**
Pre-Sales에서도 자료 조사, 고객 요구사항 정리, 제안 초안 작성, PoC 준비 같은 업무를 매번 새로 하는 것이 아니라 AI를 업무 흐름에 묶어 재사용 가능하게 만드는 방향이 중요해질 것 같습니다.

### 2. PRD
- **간단 설명**
PRD(Product Requirements Document)는 제품이나 기능의 목적, 문제, 요구사항, 성공 기준을 정리한 문서입니다. AI 시대에는 무엇을 만들지보다 **무엇을 해결해야 하는지 명확히 정의하는 능력**이 더 중요해진다고 생각합니다.

- **관련 AI 용어**
  - **Problem Framing**: 문제를 명확히 정의하는 과정
  - **Requirement**: 충족해야 하는 요구사항
  - **Success Criteria**: 결과를 판단하는 기준

- **Pre-Sales 직무 관점**
Pre-Sales는 고객의 요구를 단순 접수하는 역할이 아니라, 고객 문제를 구조화하고 기술 관점에서 다시 정의하는 역할을 많이 합니다. 그래서 PRD적 사고는 고객 요구를 명확히 번역하는 데 매우 중요합니다.

### 3. Ralph
- **간단 설명**
Ralph는 한 번 답하고 끝나는 것이 아니라, 목표가 달성될 때까지 반복적으로 검토하고 개선하는 실행 방식의 상징으로 이해했습니다.

- **관련 AI 용어**
  - **Iteration**: 반복 개선
  - **Feedback Loop**: 결과를 다시 반영하여 품질을 높이는 구조
  - **Agentic Execution**: 목표 달성 중심의 자율적 실행

- **Pre-Sales 직무 관점**
실제 Pre-Sales 업무도 한 번의 제안으로 끝나는 경우보다, 고객 질문과 피드백을 반영하며 제안 방향을 계속 수정하는 경우가 많습니다. AI를 활용하더라도 결국 중요한 것은 반복 개선과 설득의 흐름이라고 생각합니다.

### 4. Domain
- **간단 설명**
도메인은 특정 산업, 고객 환경, 업무 맥락에 대한 이해를 의미합니다. AI가 보편화될수록 일반적인 정보보다 **도메인 맥락에 맞는 해석**의 가치가 더 커진다고 생각합니다.

- **관련 AI 용어**
  - **Domain Knowledge**: 특정 산업과 업무에 대한 전문 지식
  - **Context**: 판단에 필요한 배경 정보
  - **RAG**: 외부 지식과 문서를 참고해 답변 정확도를 높이는 방식

- **Pre-Sales 직무 관점**
기술 자체를 아는 것만으로는 차별화가 어려워지고, 고객 산업과 조직 상황을 이해한 뒤 그에 맞게 기술을 설명하고 설계하는 능력이 더 중요해질 것 같습니다.

### 5. Tmux
- **간단 설명**
Tmux는 여러 작업 창을 병렬로 관리할 수 있게 해 주는 도구입니다. 저는 이를 단순 터미널 도구라기보다, 여러 작업을 동시에 다루고 맥락을 유지하는 실무 방식의 상징으로 보았습니다.

- **관련 AI 용어**
  - **Parallel Work**: 여러 작업을 동시에 진행하는 방식
  - **Context Switching**: 작업 맥락을 전환하는 과정
  - **Multiagent Collaboration**: 여러 작업 단위가 병렬적으로 협력하는 구조

- **Pre-Sales 직무 관점**
Pre-Sales는 고객 미팅 준비, 내부 기술 확인, 경쟁사 비교, 제안 문서 작성이 동시에 진행되는 경우가 많습니다. 앞으로는 이런 병렬 업무를 AI와 함께 더 효율적으로 운영하는 능력이 중요해질 것 같습니다.

## 3. 왜 이 키워드를 선택했는가

### ❓ 왜 이 키워드를 선택했는가?
이번에 선택한 키워드는 단순히 화려한 기술 용어라서가 아니라, **AI 시대에 실제로 일하는 방식이 어떻게 바뀌는지**를 보여준다고 생각해서 선택했습니다.

특히 Pre-Sales 관점에서는 직무가 완전히 다른 것으로 바뀐다기보다, 기존에 강점으로 여겨졌던 **기술 정보 비대칭**이나 **기술적 헤지**의 매력이 예전보다 낮아질 수 있다고 느꼈습니다. 고객도 AI를 통해 기본적인 기술 조사와 비교를 훨씬 빠르게 할 수 있기 때문입니다.

그래서 앞으로는 단순히 많이 아는 사람보다,
- AI를 업무 흐름에 잘 묶어 실행하는 사람(Harness)
- 고객 문제를 명확히 정의하는 사람(PRD)
- 피드백을 반영해 계속 개선하는 사람(Ralph)
- 산업과 고객 맥락을 정확히 읽는 사람(Domain)
- 동시에 여러 흐름을 효율적으로 다루는 사람(Tmux)
이 더 경쟁력이 있을 것이라고 생각했습니다.

### ❓ 이 기술이 내 직무와 어떤 부분에서 연결되는가?
- **Harness**는 AI를 실제 제안·조사·정리 업무에 재사용 가능한 방식으로 연결하는 데 필요합니다.
- **PRD**는 고객 요구를 문제 정의와 성공 기준 중심으로 재정리하는 데 연결됩니다.
- **Ralph**는 제안과 설득 과정을 반복 개선하는 Pre-Sales 업무 흐름과 연결됩니다.
- **Domain**은 기술 설명보다 고객 산업과 환경을 해석하는 역량과 직접 연결됩니다.
- **Tmux**는 여러 이해관계자와 여러 작업 흐름을 병렬로 다루는 실무 방식과 연결됩니다.

### ❓ 이 기술이 확산되면 내 직무의 어떤 업무가 변화할 것 같은가?
기본 제품 설명, 경쟁사 비교, 1차 자료 조사, 초안 문서 작성 같은 업무는 AI를 통해 더 빨라지고 표준화될 가능성이 큽니다.

반면에 더 중요해질 업무는 다음과 같다고 생각합니다.
- 고객의 진짜 문제를 정의하는 일
- 고객 상황에 맞게 기술을 번역하는 일
- 제안 방향을 반복적으로 다듬는 일
- 여러 부서와 고객 사이에서 신뢰를 형성하는 일
- 여러 흐름의 업무를 동시에 정리하고 우선순위를 잡는 일

즉, 제 직무는 사라지기보다 **기술 설명 중심 역할에서 고객 맥락 기반 의사결정 지원 역할로 더 선명해질 것**이라고 생각합니다.
