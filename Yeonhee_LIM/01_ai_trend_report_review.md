# 2 Week. AI / IT 트렌드 탐색 및 키워드 선정

> **키워드** : Sovereign AI · Agentic AI · Data Observability

> **선정 이유** : 인프라 → 자동화 → 품질 보장의 흐름으로 이어지는 AI 파이프라인의 핵심 요소

## 1. Sovereign AI

### 1-1. 설명

특정 국가나 조직이 AI 모델, 학습 데이터, 컴퓨팅 인프라를 외부에 의존하지 않고 자국·자사 내에서 직접 구축·통제하려는 흐름

### 1-2. 관련 AI 용어

| 용어 | 설명 |
|------|------|
| **Private AI Infrastructure** | 외부 클라우드가 아닌, 조직이 직접 소유·운영하는 AI 전용 인프라 |
| **On-premises LLM** | 클라우드가 아닌 자체 서버에서 운영하는 대규모 언어 모델 |

### 1-3. 왜 지금 부상하고 있는가

- 2025~2026년 **EU AI Act**가 본격 시행되면서, 데이터가 어디에 저장되고 어디에서 학습되는지가 법적 이슈가 됨
- **Gartner**는 2027년까지 35%의 국가가 지역 특화 AI 플랫폼에 투자할 것으로 전망
- **IDC**에 따르면 글로벌 소버린 클라우드 시장은 2027년까지 **2,585억 달러**에 도달할 전망

### 1-4. 왜 이 키워드를 선택했는가
 EU AI Act 같은 규제가 현실이 된 시점에서, 데이터의 물리적 위치와 법적 관할을 이해하는 것은 기술적 선택이 아니라 **필수 요건**이 되고 있다. Sovereign AI 인프라가 구축되면, 그 위에서 데이터를 수집·처리·운영하는 파이프라인 설계 수요도 함께 늘어날 것이다.

## 2. Agentic AI

### 2-1. 설명

- AI가 스스로 **계획을 세우고, 실행하고, 결과를 평가**하는 자율 AI 에이전트
- Multi-Agent 시스템으로 협업해 복잡한 비즈니스 워크플로우를 자동화
- 단순 질의응답을 넘어, 데이터 수집·전처리·적재·모니터링을 에이전트가 오케스트레이션하는 방향으로 **파이프라인 패러다임이 변화**

### 2-2. 관련 AI 용어

| 용어 | 설명 |
|------|------|
| **LAM** (Large Action Model) | 언어를 이해하는 것을 넘어, 실제로 행동(클릭, API 호출 등)을 수행하도록 학습된 모델 |
| **Multi-Agent System** | 여러 AI 에이전트가 역할을 분담해 복잡한 작업을 협업 수행하는 구조 |

### 2-3. 왜 지금 부상하고 있는가

- GPT-4, Gemini, Claude 등이 **API와 DB를 직접 조작**할 수 있게 되면서, 에이전트 패러다임이 실용화
- **McKinsey**에 따르면 이미 23%의 조직이 에이전틱 AI를 본격 확장 중이며, 39%가 실험 단계

### 2-4. 왜 이 키워드를 선택했는가
"자동화를 넘은 자율성"
- AI가 잘하는 건 결국 반복되는 패턴을 학습해서 자동으로 처리하는 것 → 에이전트가 보는 데이터가 틀려있으면 판단도 틀린 채로 끝까지 실행 → 데이터 환경 자체를 설계하는 것이 중요하다.

## 3. Data Observability

### 3-1. 설명

데이터 파이프라인의 건강 상태를 **실시간으로 모니터링**하고, 데이터의 **신선도·볼륨·스키마·분포·리니지** 등 5가지 핵심 지표를 추적하여 이상 징후를 조기에 감지하는 실천

### 3-2. 관련 용어

| 용어 | 설명 |
|------|------|
| **Schema Drift** | 업스트림 소스의 스키마가 예고 없이 변경되는 것. 파이프라인 장애의 주요 원인 ||
| **Data Lineage** | 데이터가 소스에서 목적지까지 어떤 경로로 이동·변환되었는지 추적하는 것 |
| **Universal Semantic Layer** | 데이터를 비즈니스 언어로 정의하는 층으로, Observability와 결합하여 데이터가 비즈니스 맥락에 맞게 흐르는지 검증하는 도구 |

### 3-3. 왜 지금 부상하고 있는가

- AI 시대에 모델의 품질은 곧 데이터의 품질! "Garbage in, garbage out"이 자율 에이전트 시대에는 훨씬 더 치명적일 것.
- **Gartner**에 따르면 AI 에이전트가 물리 환경에서 생성하는 데이터량이 폭증하면서, 2030년까지 **Universal Semantic Layer**가 핵심 인프라가 될 전망

### 3-4. 왜 이 키워드를 선택했는가

아무리 잘 만든 파이프라인도 모니터링 없이는 **조용히 고장 난다**.

> **데이터 다운타임** : 데이터가 누락되거나, 이상한 값이 들어오거나, 스키마가 바뀌었는데 아무도 모르는 시간대

이 시간이 길어질수록 후속 분석과 AI 모델의 결과의 안정성이 떨어진다.
특히 Agentic AI 시대에는 에이전트가 잘못된 데이터를 기반으로 **자율적으로 행동**할 수 있기 때문에, 데이터의 상태를 실시간으로 파악하는 것은 선택이 아니라 **안전장치**라고 생각했다.

## 4. 나의 산업과 직무 정의

**신뢰할 수 없는 데이터가 잘못된 판단을 만들어내는 문제**를 해결하여, 여러 소스에 흩어진 데이터를 신뢰할 수 있는 형태로 정제하고, 누구나 안심하고 사용할 수 있도록 적시에 전달하고자 한다. **데이터가 끊기지 않고 정확하게 흐르는 파이프라인**을 설계·구축·운영하는 것.

### 4-1. 키워드들이 내 산업과 직무에 미치는 영향
- Sovereign AI (인프라의 변화): 클라우드 네이티브 환경을 넘어, 폐쇄망이나 하이브리드 환경에서도 유연하게 동작하는 데이터 파이프라인 설계 역량이 요구됨. 오픈소스 기술 생태계에 대한 깊은 이해가 필수.

- Agentic AI (워크플로우의 변화): 파이프라인의 장애 복구나 쿼리 최적화를 에이전트가 수행하게 됨. 에이전트가 데이터를 잘 이해할 수 있도록 시맨틱 레이어와 온톨로지를 설계하는 역할로 확장됨.

- Data Observability (신뢰의 변화): 사후 대응 방식의 모니터링에서 이상 징후를 감지해 미리 차단하는 방식으로 변화. 데이터 품질 사고가 발생했을 때 에이전트의 실행을 즉시 중단시키는 가드레일 로직을 파이프라인에 이식해야 함.
