# AI Playbook

> 데이터엔지니어 취준생의 AI 활용 원칙·도구 전략·실행 시스템 

---

## 1. 왜 만들었나

**이전 방식의 한계:**

| 원인 | 문제 |
|------|------|
| 프롬프트 문제 | "~해줘" 형태로는 AI가 내 상황을 파악 불가 |
| 검증 문제 | AI 결과물을 그대로 수용, 출처가 비공개인 링크도 있었음 |
| 습관 문제 | 브라우저 탭으로 오가며 결과물이 대화창 안에만 남음. 비슷한 작업을 매번 처음부터 시작 |

**개선 원칙:**
1. 맥락을 파일로 관리하고 AI에 자동으로 주입한다 → `ABOUT-ME/about-me.md`
2. 결과물은 Git에 아카이빙한다 — 저장까지가 작업의 완료다
3. 반복 작업은 에이전트 파이프라인으로 자동화한다

## 2. 도구별 역할 분담

| 도구 | 역할 | 사용 시점 |
|------|------|----------|
| **Claude Code** | 파일 맥락 기반 작업, 에이전트 파이프라인 실행, 코드 작성 | 취업 준비 전 단계, ETL 스크립트, 구조화된 문서 생성 |
| **Paperclip** | 멀티에이전트 GUI 오케스트레이션 | 에이전트 5개를 파이프라인으로 시각적으로 실행 |

## 3. 구축한 시스템 — 채용 파이프라인

### 전체 흐름

```
기업 공고 발견
  → /company-analysis   (기업 뉴스·재무·JD 분석)
  → 결과 검토
  → /write-coverletter  (자소서 초안 생성)
  → 자소서 사실 확인 후
  → 병렬: /coding-test + /tech-interview + /personality-interview
  → 결과물 Git 커밋 → OUTPUTS/ 최종본 저장
```

### 스킬 명세

| 스킬 | 인풋 | 저장 위치 |
|------|------|---------|
| `/company-analysis` | 기업명, 채용공고 URL, JD | `companies/[기업명]/analysis-YYYY-MM-DD.md` |
| `/write-coverletter` | 기업명, 자소서 항목·글자 수 | `coverletters/[기업명]/draft-YYYY-MM-DD.md` |
| `/coding-test` | 기업명, 유형(python\|sql\|both), 난이도(junior\|mid) | `coding-tests/[기업명]/problems-YYYY-MM-DD.md` |
| `/tech-interview` | 기업명, 모드(generate\|simulate\|review) | `interview-prep/[기업명]/tech-YYYY-MM-DD.md` |
| `/personality-interview` | 기업명, 모드(generate\|simulate\|review) | `interview-prep/[기업명]/personality-YYYY-MM-DD.md` |

### Paperclip을 쓴 이유

1. **병렬 실행 불가** — Claude Code는 대화 한 턴에 한 스킬만 실행된다. 코딩테스트·기술면접·인성면접은 서로 의존성이 없으므로 동시에 실행하면 시간을 줄일 수 있다.
2. **실행 상태 가시성 없음** — 어느 에이전트가 어디까지 완료됐는지 대화창으로는 추적이 어렵다.

Paperclip은 에이전트를 노드로 연결해 파이프라인 흐름을 GUI에서 보여주고, 완료된 단계와 진행 중인 단계를 구분해준다. 기업 수가 늘어날수록 이 가시성이 중요해진다.

### Paperclip 에이전트 구성


| 에이전트 | 모델 | 모델 선택 이유 |
|---------|------|--------------|
| 기업분석 | claude-sonnet-4-6 | 웹검색·재무 조회·JD 심층 분석 등 복잡한 추론이 필요 |
| 자소서 | claude-sonnet-4-6 | 실제 경험을 STAR로 구조화하는 글쓰기 품질이 결과물에 직결 |
| 코딩테스트 | claude-haiku-4-5 | 문제 생성·코드 리뷰는 정해진 패턴 작업 — 빠른 응답속도와 낮은 비용이 우선 |
| 기술면접 | claude-haiku-4-5 | 예상 질문 생성·모의 면접은 반복 패턴 작업, 턴 수가 많아 비용 절감 효과 큼 |
| 인성면접 | claude-haiku-4-5 | 동일한 이유. 기술면접과 병렬로 돌리면 Sonnet 대비 비용이 약 1/5 |

> **모델 선택 기준 요약**: 창의적 판단·긴 문맥 통합이 필요한 앞단(기업분석·자소서)은 Sonnet, 구조화된 패턴 반복 작업인 뒷단(코딩테스트·면접)은 Haiku로 비용과 속도를 최적화했다.

## 4. 단계별 프롬프트

### [1단계] 기업분석 — `/company-analysis`

```
너는 15년 경력의 채용 전문 컨설턴트야.
나는 데이터엔지니어 직무에 지원하려는 취준생이야.
아래 채용공고와 내 이력서를 바탕으로 웹검색을 병행하여 6단계 심층 분석을 진행해줘.

[채용공고 URL] / [JD 전문] / [내 이력서]

0단계: 기업 정밀 스캔
  (A) 최신 뉴스 (최근 6개월) — 주요 이슈, 기술 투자 동향
  (B) 재무제표 — 최근 3년 매출/영업이익, 부채비율 (DART 또는 언론 대체)
  (C) 기업 포지셔닝 — 성장 동력, 데이터 조직 역할

1단계: 직무 해부학 — JD 표면 + 숨겨진 핵심 역량 10가지
2단계: 키워드 크랙킹 — 필수/우대 분리, 자소서·면접 필수 언급 키워드
3단계: 이력서 매칭 분석 — 강점 / Gap / 보완 전략
4단계: 인재상 리버스 엔지니어링 — 예상 합격자 프로필
5단계: 채용 프로세스별 준비 로드맵 (자소서·코딩테스트·면접)
```

---

### [2단계] 자소서 — `/write-coverletter`

**Step 1. 항목 분석**
```
너는 데이터엔지니어 채용 전문 컨설턴트야.
아래 자소서 항목들이 평가하려는 핵심 역량을 정리해줘.

- 지원 기업: [기업명]
- 자소서 항목: [항목 리스트]

각 항목에 대해:
1) 평가하려는 핵심 역량
2) JD에서 연결되는 키워드
3) 쓰면 좋은 경험 유형
```

**Step 2. 초안 작성**
```
너는 데이터엔지니어 직무에 특화된 자소서 컨설턴트야.
아래 조건으로 STAR 기법 초안을 작성해줘.

- 지원 기업: [기업명]
- 직무 키워드: [JD 핵심 역량 3~5개]
- 자소서 항목: [질문]
- 글자 수 제한: [N자]

[내 경험 소재]
- 프로젝트명 / 내 역할 / 사용 기술 / 문제 상황 / 해결 과정 / 성과(수치)

요구사항: 미사여구 없이 간결하게, 직무 연결점 자연스럽게 포함
```

**Step 3. 첨삭**
```
아래 자소서를 데이터엔지니어 채용 관점에서 첨삭해줘.
1) JD 키워드 정합성  2) 구체성(수치·기술명)  3) STAR 논리 흐름
4) 불필요한 표현 제거  5) 글자 수 [N]자 준수
```

---

### [3단계] 코딩테스트 — `/coding-test`

**Step 0. 기업별 출제 경향 분석**
```
나는 Data Engineer 직무로 {기업명}에 지원하는 취준생이야. 언어는 Python만 사용해.

1. {기업명} 코딩테스트 개요 (플랫폼, 문제 수, 제한 시간)
2. 출제 유형 분석 (빈출 순, 각 유형별 빈출도·난이도·핵심 알고리즘)
3. {기업명} 맞춤 전략 (Python 유의사항, 시간 배분, 부분 점수)
4. 빈출 유형별 Python 필수 템플릿 skeleton 코드

| 유형 | 빈출도 | 난이도 | 핵심 알고리즘 | Python 핵심 라이브러리 |
```

**Step 1. 힌트 (막혔을 때만)**
```
아래 문제에서 막힌 부분에 대해 접근 방향만 힌트로 줘.
정답 코드는 절대 주지 마. Python 기준으로 설명해줘.

- 유형: [ex. 그래프 탐색 / DP]
- 막힌 부분: [구체적 상황]
- 내 현재 접근: [ex. 완전탐색으로 시도했지만 시간 초과]
```

**Step 2. 코드 리뷰**
```
아래 Python 코드를 리뷰해줘. 나는 데이터엔지니어 취준생이야.
1) 시간·공간 복잡도  2) 엣지 케이스 누락  3) 가독성·변수명
4) 더 효율적인 Python 접근법 비교

[문제 요약]: / [내 풀이]: (코드)
```

### [4단계] 기술면접 — `/tech-interview`

**generate 모드 — 예상 질문 생성**
```
너는 [기업명]의 데이터엔지니어 채용 면접관이야.
JD와 내 이력서를 바탕으로 면접 예상 질문을 생성해줘.

[JD 핵심 키워드] / [내 이력서 주요 경험]

1) 기술 면접 질문 5개 (꼬리 질문 포함)
2) 프로젝트 경험 기반 질문 3개
3) 인성/조직 적합성 질문 3개
```

**simulate 모드 — 모의 면접**
```
너는 시니어 데이터엔지니어 면접관이야. 모의 기술 면접을 진행해줘.
- 질문 하나씩 → 내 답변 → 점수(1~10) + 보완점 + 꼬리 질문
- 마지막에 전체 총평

직무: 데이터엔지니어 / 난이도: 주니어
주제: [ex. ETL 파이프라인 설계 / Airflow DAG / 파티셔닝 전략]
```

**review 모드 — 답변 구조화**
```
아래 면접 질문에 대한 내 답변을 90초 이내로 구조화해줘.
- STAR 기법 / 핵심 키워드 앞에 배치 / 불필요한 서론 제거 / 수치·기술명 포함

[질문]: / [내 답변 초안]:
```

## 5. AI 활용 원칙

1. **맥락부터 전달한다** — `ABOUT-ME/about-me.md`가 공통 맥락 파일이다. 모든 스킬이 이 파일을 먼저 읽는다.
2. **작업별로 도구를 나눈다** — 하나의 도구에 모든 것을 맡기지 않는다.
3. **AI 출력은 초안이다** — 사실 확인 → 수정 → Git 저장까지가 작업의 완료다.
4. **AI가 만든 경험은 쓰지 않는다** — 자소서·면접 답변에는 `ABOUT-ME/about-me.md`의 실제 경험만 사용.

## 6. 검증 루틴

| 검증 유형 | 방법 |
|----------|------|
| **사실 검증** | AI가 언급한 수치·통계는 원문 출처 반드시 확인 |
| **기술 검증** | AI가 만든 코드는 테스트 데이터로 직접 실행 |
| **자소서 검증** | "내 실제 경험인가?" 체크. AI가 생성한 경험 문장은 제거 |
| **면접 답변 검증** | 90초 이내 분량인가, STAR 구조인가, 기술 용어가 정확한가 |
| **코딩테스트 검증** | AI 코드를 외우지 않고 원리를 설명할 수 있는가 |
