# 4 Week. AI 활용 성공 사례 분석

## 1. 사례 기본 정보

- **기업**: 우아한형제들
- **사례**: 리뷰 데이터 기반 개인화 메뉴 추천 AI + 허위 리뷰 탐지 시스템
- **선택 이유**: 이 사례는 AI 모델의 성능이 아닌, **데이터 품질이 비즈니스 성과를 직접 만들어낸 구조**를 보여주기 때문이다.

## 2. 문제 정의 

- **핵심 문제**: 추천 모델이 학습하는 리뷰 데이터에 허위·조작 리뷰가 혼재하여, **데이터 신뢰도가 낮아 추천 품질을 담보할 수 없는 구조**였다. 이로 인해 사용자가 원하는 메뉴를 빠르게 찾지 못해 이탈률이 높아지는 비즈니스 문제로 이어졌다.
- **기존 방식의 한계**:
  - 허위 리뷰를 사람이 수동으로 신고·검수하는 방식은 **확장성** 한계
  - 데이터 품질 검증 없이 원시 데이터가 그대로 추천 모델에 유입되는 구조 → **오염된 데이터 기반 학습**
  - 소비자가 직접 탐색하는 **수동적 구조** → 상황·기분에 맞는 메뉴를 찾기까지 시간 소요

## 3. 해결 방식

- **AI 적용 단계**: 데이터 품질 관리(전처리) → 분석(모델 학습) → 서빙(실시간 추천)

### AI 도입 Before / After

| 구분 | Before | After |
|------|--------|-------|
| 메뉴 탐색 | 사용자가 카테고리를 직접 탐색 | AI가 상황·문맥 맞춤 추천 |
| 리뷰 품질 관리 | 사용자 신고 → 수동 검수 | AI 자동 탐지 + 필터링 |
| 추천 데이터 기반 | 단순 인기순·거리순 | 행동 데이터 핵심 피처 + 리뷰 보조 피처 학습 |

## 4. 성과
  - 추천 기능 적용 영역에서 **주문 전환율 80% 증가**
  - 허위 리뷰 제보 건수 **83% 감소**
- 이유 : 추천 모델 자체의 고도화도 중요하지만, 그 모델이 학습하는 **데이터의 품질을 보장하는 전처리 파이프라인**이 선행되지 않았다면 동일한 성과를 내기 어려웠을 것이다. 허위 리뷰 탐지라는 품질 게이트가 데이터 신뢰도를 확보하고, 그 위에서 행동 데이터 기반 추천이 정확도를 높인 구조다.


## 5. 조건 / 리스크

- **성공 조건**:
  -  **대규모 데이터셋** — 패턴 학습의 전제 조건
  - 허위 리뷰 탐지 모델의 정밀도가 충분히 높아야 정상 리뷰를 오삭제하지 않음
  - 실시간 추천을 위한 **저지연 서빙 인프라** 구축
- **리스크**:
  - 허위 리뷰 탐지의 오탐 시 정상 리뷰 삭제 → 사용자·업주 불만
  - 추천 편향: 특정 업체에 추천이 집중되면 플랫폼 공정성 이슈 발생
  - 리뷰 패턴이 변화(신조어, 이모지 등)하면 모델 재학습 필요

## 6. 내 업무 적용

- **핵심 원리**:
  1. AI 성과의 출발점은 모델이 아니라 **데이터 품질 관리**이다
  2. 데이터 검증 레이어는 파이프라인 설계 시점부터 내재화해야 한다

- **적용 전략**: 배민의 허위 리뷰 탐지처럼, ETL/ELT 파이프라인의 Transform → Load 사이에 **데이터 품질 게이트**를 배치한다. 이상치 탐지·중복 제거·포맷 검증 등의 품질 검증 로직을 자동화하고, 임계값 미달 시 파이프라인을 중단하거나 알림을 보내는 구조를 설계한다.

## 7. 느낀 점 / 내 생각

배달의민족 사례에서 주목한 점은 추천 모델의 성과가 **데이터 품질에서 출발했다**는 구조다. 허위 리뷰 탐지라는 품질 게이트가 파이프라인에 내재화되어 있었기 때문에, 추천 모델이 신뢰할 수 있는 데이터 위에서 학습할 수 있었다.

파이프라인을 "빠르게 만드는 것"에 집중하기 쉽지만, 실제로 비즈니스 임팩트를 만드는 것은 **깨끗한 데이터가 흐르도록 보장하는 품질 게이트 설계**라는 점이다.
