# 3 Week. 내 직무에서의 AI 기회 분석

> **직무** : Data Engineer

> **분석 관점** : 직무 성과를 극대화하기 위해 AI가 가장 먼저 들어가야 할 업무


## 1. 직무의 가치 정의

### 1-1. 직무 정의
원천 데이터를 수집·변환·적재하는 파이프라인을 설계하고 운영하여, 데이터를 안정적으로 사용할 수 있도록 인프라를 구축

---

### 1-2. 조직(회사)에 주는 가치

| 구분 | 내용 |
|------|------|
| **잘 수행되면** | **의사결정 속도 향상** : 원천 데이터가 비즈니스 지표로 변환되는 시간을 최소화하여 의사결정을 가속화한다.
| **부족하면** | **의사결정 왜곡·지연** : 지연이 크거나 품질이 낮은 데이터 때문에 잘못된 지표를 보고 판단하거나 의사결정이 늦어진다. |

---

### 1-3. 고객·서비스 측면에서의 가치

| 구분 | 내용 |
|------|------|
| **잘 수행되면** | **개인화 서비스 고도화**: 안정적인 데이터로 추천·랭킹·타겟팅 모델이 잘 동작해, 고객에게 더 관련성 높은 콘텐츠·상품을 노출할 수 있다. |
| | **서비스 안정성 및 응답성**: 트래픽·사용 로그가 실시간에 가깝게 처리되어, 장애 대응이 빠르게 이뤄져 고객이 느끼는 품질이 좋아진다. |
| **부족하면** | **추천·검색 품질 저하**: 로그 누락·중복·지연 때문에 추천·검색이 부정확해진다. |
| | **서비스 장애 체감**: 모니터링과 지표 파이프라인이 불안정하면 장애 대응이 늦어져, 응답 지연·에러 페이지로 직결된다. |


---

## 2. 직무 성과 기준

### 2-1. 정량 지표
- **Data Freshness:** 데이터의 지연 정도 측정하여 정의된 SLA 준수율를 만족했는지 평가
    - SLA(Service Level Agreement): 데이터를 언제까지 제공하겠다는 시스템 시간 기준 계약
- **Pipeline Success Rate:** 전체 파이프라인 대비 장애 발생 없이 완료된 비율.

### 2-2. 정성 지표
- **Architecture Scalability:** 데이터 모델과 파이프라인이 데이터 양의 급증이나 새로운 요구사항에도 시스템이 유연하게 대응 가능한지 여부.

## 4. AI 우선 적용 영역 선정

### 4-1. AI 적용 우선순위 업무

- ETL Pipeline 개발
- 파이프라인 모니터링/디버깅

### 4-2. 선정 이유
#### ① 반복성이 높다
- ETL: 새 테이블 추가할 때마다 "추출 → 변환 → 적재" 패턴을 반복하여 소스만 다르고 구조는 비슷하다.
- 모니터링: 매일 같은 대시보드 확인, 같은 유형의 알림 처리를 반복한다.

#### ② 실수·병목 위험이 크다
- ETL: 컬럼 매핑 실수, 타입 변환 누락 같은 사소한 실수가 데이터 전체에 영향을 끼칠 수 있다.
- 모니터링: 이상 징후를 못 잡으면 조용히 잘못된 데이터가 쌓이게 된다.

### 4-3. AI 적용 방식 및 기대효과

#### 파이프라인 모니터링 / 디버깅

| 항목 | 내용 |
|------|------|
| **활용 AI 기술** | Anomaly Detection (이상 탐지) |
| **적용 방식** | ① 파이프라인 실행 메트릭(실행 시간, 레코드 수, 리소스 사용량)을 자동 수집 → ② AI가 평소 패턴을 학습해 이상 징후를 실시간 감지 → ③ 이상 감지 시 LLM이 과거 장애 로그를 참고해 **원인 분석 리포트 자동 생성** → ④ 사전 정의된 복구 전략(재실행, 롤백 등) 자동 실행 |


| 기대효과 | 내용 |
|---------|------|
| **MTTR (복구 시간)** | 장애 복구 시간 단축|
| **다운타임** | 데이터 다운타임 감소 |


> **결론:** AI는 파이프라인 장애를 사전에 감지하고 자동 복구하는 지점에 가장 먼저 들어가야 한다. 데이터 다운타임을 줄이고, 후속 Agentic AI의 안전한 실행을 보장하는 가장 높은 ROI가 될 수 있을 것이다.
