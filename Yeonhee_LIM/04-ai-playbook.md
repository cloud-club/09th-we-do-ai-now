## 나의 AI Playbook 초안

### 1. 나의 AI 활용 원칙

1. **맥락부터 전달한다** — 역할, 배경, 제약 조건, 원하는 출력 형식을 프롬프트에 반드시 포함  
2. **작업별로 도구를 나눈다** — 하나의 도구에 모든 것을 맡기지 않는다  
3. **결과물은 반드시 검증하고 저장한다** — AI 출력은 초안이다. 사실 확인 → 수정 → Git 저장까지가 완료

### 2. 도구별 역할 분담

| 도구 | 역할 | 사용 시점 |
|------|------|----------|
| **ChatGPT** | 빠른 개념 확인, 브레인스토밍, 비교 대상 | 새로운 기술 키워드 탐색, 아이디어 발산 |
| **perplexity** | 기업 재무재표, 현직자 인터뷰, 블라인드 등 자료 수집 | 기업 분석, 자소서 작성 및 면접 준비 |
| **Claude** | 긴 문서 분석, 구조화된 글쓰기, 자소서/면접 답변, 팩토링, 파일 맥락 기반 작업 | 코딩 과제, ETL 스크립트 작성, 기업 분석, 면접 준비, 보고서 작성 |

### 3. 표준 사용 흐름

```
업무 입력 (JD 분석 / 코드 작성 / 면접 준비)
  → 적합한 도구 선택 (도구별 역할 분담 기준 참고)
  → 맥락 + 프롬프트 템플릿으로 AI에 요청
  → 결과물 1차 확인 (핵심 요구사항 충족 여부)
  → 사실·기술 검증 (공식 문서, 실행 테스트)
  → 수정·보완 후 마크다운으로 정리
  → Git 커밋으로 아카이빙
```

### 4. 검증 루틴

| 검증 유형 | 방법 |
|----------|------|
| **사실 검증** | AI가 언급한 수치·통계는 원문 출처를 반드시 확인 |
| **기술 검증** | AI가 만든 코드는 테스트 데이터로 검증 |
| **문서 검증** | 자소서·면접 답변은 "내 경험인가?" 기준으로 재확인. AI가 지어낸 경험을 그대로 사용하지 않음 |
| **실행 가능성 검증** | 제안된 아키텍처나 전략이 현재 나의 기술 스택과 리소스로 실현 가능한지 판단 |
