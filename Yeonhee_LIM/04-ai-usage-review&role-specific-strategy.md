# 5 Week. 나의 AI 사용 점검 및 직무 맞춤 적용 전략 설계


## 1️. 현재 나의 AI 사용 방식 점검

### 1-1. 현재 사용 중인 도구

| 도구 | 주 사용 목적 |
|------|-------------|
| **ChatGPT** | 빠른 개념 확인, 브레인스토밍, 비교 대상 |
| **Claude** | 구조화된 글쓰기, 자소서/면접 답변, 리팩토링, 파일 맥락 기반 작업 |
| **perplexity** | 기업 재무재표, 현직자 인터뷰, 블라인드 등 자료 수집 |
| **manus**| github, google drive 연동 + 실행 |

### 1-2. 자주 사용하는 패턴

- **내가 가장 자주 하는 AI 활용 방식**: 프로젝트 사용 외에 브라우저로 ChatGPT/Claude 탭을 열어 놓고, 코드 에디터와 `Alt+Tab`으로 오가며 사용
- **현재 방식의 한계**:
  - 프로젝트 폴더별로 구분하지 않아 맥락이 섞임
  - 대화가 길어지면 이전 맥락을 잃어버려 같은 설명을 반복함
  - 결과물이 브라우저 대화창 안에만 존재하여 나중에 찾기 어려움

## 2️. 지금까지의 사용에서 아쉬웠던 점 정리

### 2-1. 업무 연결성 저하

AI 결과물을 복사해서 따로 정리하는 과정에서 시간이 소모가 큼

### 2-2. 단발성 사용
- "일단 빠르게 답을 얻자"는 생각
- 결과물을 별도로 아카이빙하지 않아, 비슷한 질문을 할 때마다 처음부터 다시 시작


## 3. 실패 패턴 분석

### 3-1. 실패 원인 정리

| 원인 유형 | 분석 |
|----------|------|
| **프롬프트 문제** | "~해줘" 형태의 프롬프트로는 AI가 내 상황을 알 수 없었음 |
| **검증 문제** | AI 결과물을 그대로 수용. 가끔씩 출처 들어가보면 링크 비공개 처리 |
| **사용 습관 문제** | 자동화 가능한 반복 작업을 수동으로 계속 수행 |

### 3-2. 앞으로의 개선 원칙

1. **프롬프트 자동화 및 구체화** — 프롬프트에 역할·배경·원하는 형식을 명시하고, 필요한 참조 문서를 함께 입력한다
2. **결과물을 반드시 검증하고 아카이빙한다** — AI 출력은 초안이며, 사실 확인 → 수정 → 저장까지가 하나의 워크플로우다


## 4. 내 직무에 맞는 AI 적용 전략 설계

### 4-1. 데이터엔지니어

| 업무 | AI 적용 방식 | 기대 효과 |
|------|------------|----------|
| **ETL 코드 작성 및 리뷰** | 쿼리 최적화, 에러 디버깅 | 코드 품질 향상 |
| **파이프라인 모니터링 및 알림설정** | 로그 분석 자동화, 이상 징후 요약, 장애·이상 감지 시 Slack/Email 알림 규칙을 AI로 자동 생성 | 장애 원인 파악 시간 단축, 장애 인지 → 대응까지의 리드타임 단축 |
| **데이터 품질 검증** | 검증 규칙 자동 생성, 스키마 변경 감지 | 수동 검수 부담 감소 |

**활용 방식**:
- **CLI/API 자동화** — 프로젝트 폴더에서 직접 코드 작성·리팩토링. 파일 맥락을 자동으로 잡아줌
- **문서/지식 아카이빙 연계** — AI 결과물을 마크다운으로 정리하여 반복 사용 가능한 지식 베이스 구축

### 4-2. 취준생

| 업무 | AI 적용 방식 | 기대 효과 |
|------|------------|----------|
| **JD 분석 및 역량 매칭** | JD에서 필수/우대 역량 추출 → 내 경험과 Gap 분석 | 기업별 맞춤 준비 시간 단축|
| **기술 면접 시뮬레이션** | 예상 질문 생성 → 답변 작성 → 피드백 루프 반복 | 혼자 준비하는 것보다 밀도 높은 연습 |
| **자소서/면접 답변 구조화** | STAR 기법 적용, 직무 키워드 매칭 | 답변 품질 향상, 일관된 스토리라인 확보 |

**활용 방식**:
- **Agent/Workflow 확장** — JD 수집 → 분석 → 역량 매칭 → 면접 질문 생성을 자동 파이프라인으로 구축
