## 3. 키워드별 간단 정리

### 3.1 AIOps

- **간단 설명**  
전통적으로 사람이 대시보드·알람·티켓을 넘나들며 하던 **상관분석·노이즈 제거·우선순위화**에 ML·규칙 엔진을 결합하고, Runbook·챗봇·에이전트와 연계해 **사건 분류·초기 대응**을 가속하는 운영 패러다임이다. “AI가 운영을 대체”라기보다 **데이터가 많은 운영 업무에서 패턴 인식과 반복 판단을 보조**하는 쪽에 가깝다.
- **관련 AI 용어 (간단)**  
  - **ML/지도·비지도 학습**: 라벨된 장애 데이터로 분류하거나, 정상 패턴만으로 이상치 탐지  
  - **시계열 예측**: 용량·트래픽·에러율 예측으로 사전 알람  
  - **NLP**: 로그·티켓 텍스트 요약, 유사 사건 검색  
  - **RCA(Root Cause Analysis) 지원**: 이벤트 체인·의존성 그래프 기반 후보 원인 랭킹(제품마다 규칙+학습 혼합)

### 3.2 Observability

- **간단 설명**  
“모니터링”이 주로 미리 정한 지표 위주라면, Observability는 **질문에 답하기 위해** 로그·메트릭·트레이스를 연동해 원인 추적 가능성을 높이는 것이다. AIOps 모델의 **입력 품질·컨텍스트**를 결정한다.
- **관련 용어**  
**메트릭·로그·분산 트레이싱**, **서비스 맵/의존성**, **SLO/에러 버짓** 등

### 3.3 Anomaly detection

- **간단 설명**  
기준선에서 벗어난 패턴을 자동 표시해 **알람 피로**를 줄이고 **미탐·오탐** 균형을 맞추려는 기법. 금융·클라우드 운영에서는 배치 지연, API 오류율, 비정상 트래픽 등에 적용된다.
- **관련 용어**  
**기준선(baseline)**, **계절성**, **다변량 이상 탐지**, **오탐 제어(false positive)**

## 4. 왜 이 키워드를 선택했는가

### 4.1 선택 이유

- 클라우드·하이브리드 환경에서 **데이터 소스와 알람이 기하급수로 늘어나** 사람만으로는 상관관계 파악과 우선순위 선정에 한계가 있다는 점이 트렌드 리포트와 실무 체감 모두에서 공통된다.  
- **AIOps**는 “운영 데이터 + AI”가 만나는 대표 축이라, 앞으로 **로그·배치·메트릭 분석을 강화하는 직무**와 직접 맞닿아 있다.  
- **Observability**와 **이상 탐지**는 AIOps를 떼어놓고 이야기하기 어려운 전제·핵심 기술이라 함께 묶어 이해하는 것이 낫다고 보았다.

### 4.2 내 직무와의 연결

- 직무 맥락상 **AWS 기반 설계·변경, 모니터링·백업, 로그·배치 기반 정합성·이상 확인, 장애·이슈 대응, 비용·통제**가 겹친다.  
- AIOps는 **대량 로그·시계열·이벤트를 요약·군집·이상 표시**하는 쪽으로, 이미 하고 있는 “데이터로 건강도를 보는” 업무와 같은 축이다. 다만 **금융권 규제·감사·내부통제** 환경에서는 **모델·알람·자동 조치의 설명 가능성, 권한, 감사 추적**이 일반 SaaS AIOps 도입보다 더 중요한 제약이 된다.

### 4.3 기술이 널리 쓰이면 직무에서 어떻게 바뀔 것 같은가

- **긍정적 변화**  
  - 반복 스캔·문자열 비교 시간을 줄이고, **이상 구간 후보·우선순위**를 빠르게 얻을 수 있음  
  - 장애 시 **영향 범위·유사 과거 사례** 검색이 쉬워져 MTTR·설명 자료 작성 부담 완화 가능
- **주의·변화**  
  - **알람·자동화 정책**이 늘면서 **변경 관리·권한·승인·로그 보존** 요구가 더 엄격해질 수 있음  
  - AI 보조 결과는 **원본 로그·설정·공식 절차와 대조**하는 검증 단계가 표준이 되어, “도구 출력 그대로”보다 **증적·근거 중심 업무** 비중이 유지되거나 강화될 수 있음

## 5. 나의 산업·직무 정의

- **산업**  
금융권 IT 인프라·클라우드 운영(규제·내부통제·대외 보고와 밀접한 환경)
- **직무가 푸는 문제**  
**가용성·보안·비용·데이터 정합**을 동시에 맞추고, **금융보안원 가이드·금감원 보고·감사**가 요구하는 **운영 상태와 증빙**이 실제 AWS·운영 데이터와 맞도록 유지·설명하는 문제
- **만드는 결과물**  
**설계·적용된 통제와 운영 사실**, **로그·배치·비용·변경에 대한 설명 가능한 근거**, **점검·보고·조치가 연결된 운영 신뢰도**

**한 문장 정의**

> **금융권 규제와 내부통제 요구를 충족하는 AWS 운영·데이터 증적을 유지하면서, 로그·배치·인프라 신호로 서비스 건강도와 정합성을 판단·설명하는 클라우드·운영 엔지니어링 역할**이다. AIOps는 이 판단을 돕는 도구 축으로, 최종 책임과 대외·감사 해석은 사람이 가진다고 본다.
