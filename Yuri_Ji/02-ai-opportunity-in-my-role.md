# Week 3 과제 — 직무에서 AI가 필요한 지점 찾기

## 1. 직무의 가치 정의

### 1.1 이 직무가 만드는 가치

- **조직**  
  - 클라우드·보안·내부통제 요구의 AWS 반영, **규제·감사 대응 운영 상태 유지**  
  - **금융감독원 클라우드 이용 보고·내부 사규·안전성 확보조치** 등 **절차·문서**와 실제 운영 **정합성**  
  - 로그·배치**를 통한** 오류·이상 **조기 식별**, 수작업·점검 기반 **시스템 신뢰도 유지**  
  - 비용 **예산·사용량 가시성**(예측·통제 용이), **낭비·유휴 비용 감소**(FinOps·비용 검수 맥락)
- **서비스**  
  - **가용성·복구·통제** 측면 **기반 정렬**  
  - 데이터 정합성 **확보**, 장애·이상 **사전 탐지**, 품질·리스크 **동시 관리**
- **고객**  
직접 응대 없음. 인프라·통제 실패 또는 운영 데이터 검증 실패 시 **서비스 연속성·신뢰 훼손 위험**.

### 1.2 잘 수행될 때와 부족할 때

**잘 수행 시**

- 점검 시 근거·절차·설정 **설명 가능성**, 통제 공백·반복 질문 **감소**  
- 금감원·내부 요구 **보고·증빙** 기한·품질 **충족**, 부서 간 협업 **마찰 최소화**  
- 오류·이상 **조기 발견**, 운영 신뢰도 **상승**, 후속 장애·이슈 **감소**  
- 비용·변경 **통제**, 장애·보안 이슈 대응 **정리**

**부족 시**

- 통제 미비 지적·증빙·문서 재작업 **증가**, 규제·내부통제 리스크 **확대**  
- 보고·심의·계약 단계 **지연·보완 반복**  
- 오류 **누락·장애 연쇄**, 데이터 신뢰도 **저하**, 원인 파악 **난항**  
- 비용 누수·동일 질의·자료 요청 반복 **에 따른 비효율**

### 1.3 레퍼런스·대외 절차와의 관계

- **금융보안원 「금융분야 클라우드컴퓨팅서비스 이용 가이드」** — 설계·운영·문서 **교차 확인**, 개정 시 **변경 지점** **아키텍처·절차·증빙 반영**  
- **금융감독원 보고·내부 클라우드 이용 절차** — 과제 선정·**중요도 평가**, **CSP 건전성·안전성 평가**, **안전성 확보조치** 수립, **정보보안위원회** 심의, **계약**, **이용·보고**(사용 전후·기한), **종료**(이관·삭제·퇴출) 등 **다부서·다단계 흐름**  
- **로그·운영 증적**(로그 보관·모니터링·점검 기록 등) — 금융보안원 가이드·내부 점검·금감원 보고에서 **자주 요구되는 항목**

## 2. 직무의 성과 기준 정의

### 2.1 정량 지표 (예시)


| 영역     | 지표 예시                                                          |
| ------ | -------------------------------------------------------------- |
| 비용·인프라 | 예산 대비 비용, 이상 구간 식별·조치, 태깅·정리 **통한** 미분류 비용 감소, **비용 검수** 이슈 건수 |
| 운영·가용성 | 장애 건수·심각도, MTTR, 변경 실패율, 모니터링·알람 커버리지                          |
| 통제·보안  | 정책 위반 건수, 이슈 조치 기한 준수율, 백업·복구 점검 이행률                           |
| 감사·점검  | 조치 완료율, 재지적 감소, 증빙·회신 리드타임                                     |
| 규제·보고  | 보고·제출 **기한 준수**, 보고 패키지·체크리스트 항목 **충족률**, 보완·재요청 **감소**        |


※ 수치 **내부 목표·대시보드 기준 구체화**.

### 2.2 정성 지표

- 감사·점검: 근거 **일원화**, 반복 설명 **최소화**  
- 금감원·가이드·내부 절차: 요구 **정합성**, 대외·대내 설명 **일관성**  
- 문서·운영 절차: 절차·책임 **명확성**  
- 로그·배치 분석: 결과 **신뢰도**, 전달 **명확성**  
- 운영: 안정감, 변경·장애 시 문서·실제 **일치**  
- 협업: 사용부서·IT·정보보안·외부기관 **협업 효율**, 문제 대응 **일관성**

## 3. 직무의 업무 분해

### 3.1 주 업무 / 부 업무 / 기타

- **주**  
  - 파트 **공동 담당** 범위: AWS 계정·랜딩존·네트워크·IAM·백업·모니터링·보안 통제 **설계·적용**, 변경 관리, 장애·이슈 대응, 비용 관리  
  - **로그·배치** 분석, 운영 결과·이상 징후 확인, 이슈·원인 파악 — **운영 데이터 기반** 건강도 확인 축(DB·업무 데이터 **직접 관리**와는 구분)  
  - **감사 대응**, 인프라 **유지·변경**
- **부**  
  - 금융보안원 가이드·**금감원 보고·안전성** 관련 **문서 작성·대조·증빙** 정합성(보고·절차 **문서 작업**), 내부 기준 점검, 타 부서 요청  
  - 플랫폼·OSS(K8s, Nexus 등) **가이드·문의** 대응 일부, 도구·자동화·파이프라인  
  - 애플리케이션 관련 작업 요청 대응 : 엑셀·텍스트 가공, 스크립트(Python 등), 결과 정리·공유
- **기타**  
  - **교육·신규 서비스 정보 전파** 관련 협조  
  - 교육·회의, 공지 모니터링, 벤더 대응

### 3.2 반복·비효율·실수 위험 큰 업무

**규제·감사·보고**  
증빙·설명 자료(로그·설정·통제 매핑), **지정보고·위탁·중요도·CSP 평가·안전성** 등 **보고 패키지** 구성, 표준·운영 절차·변경 기록, 가이드·규정 **대조**(반복 독해), 개정 **변경 지점 추적**, 비용 증감 **원인·검수**  

**운영 데이터**  
대량 로그 수작업 스캔, 로그·배치 결과 대조, 엑셀 반복 확인, 결과 **해석·요약·전달**  

**공통**  
반복·시간 소모·누락·오해·표현 불일치 **실수 가능성**

## 4. AI 우선 적용 영역 선정

### 4.1 적용 전제 (역할 범위)


| 구분                              | AI에 기대는 것                                                      | 사람이 가져가는 것                          |
| ------------------------------- | -------------------------------------------------------------- | ----------------------------------- |
| **규제·대외** (금융보안원 가이드·감사·금감원 보고) | **문서 작업** 보조: 초안, 요약·목차, 대조·개정 포인트 후보, 증빙·체크리스트 초안             | “규정상 가능한가”, 대외 답변·**승인**, **최종 판단** |
| **AWS·관측 가능** (지표·로그·설정 비교적 명확) | 로그·메트릭 요약, 이상 후보, 변경 전후 diff 설명 **초안**, 자동화·보조                 | 감사·운영에 올리는 **확정 결론**, 사실 여부 **검증**  |
| **레거시·얽힘** (의존·배치·로그가 흩어짐)      | 운영 반영 **전** 얽힌 지점·영향·유의사항 **후보** 짚기(회사 **검토 중**, 도입·책임은 논의 단계) | 가동·승인 **전** 리스크 판단, 범위·책임 **결정**    |


※ 금융보안원·금감원 관련 업무는 **DB·업무 데이터를 직접 관리**하는 성격보다, **절차·보고·증빙을 맞추는 문서 작업** 비중이 크다.

### 4.2 우선순위별 적용 후보


| 순위    | 영역                                         | 선정 이유                                                        | 활용 방향                                                                      |
| ----- | ------------------------------------------ | ------------------------------------------------------------ | -------------------------------------------------------------------------- |
| **1** | (모니터링) 로그·배치 결과 분석                         | 데이터량·수작업 한계, 이상 탐지 반복, 미탐 시 장애 리스크. 로그·증적은 감사·가이드·보고와 **연계** | 패턴 분류·요약 초안, 이상 구간 후보, 기준 대비 변화 감지 후보, 자연어 요약 초안(공유 전 **사실 대조**)           |
| **2** | (문서작업) 금융보안원 가이드 + 금감원 보고·이용 절차 (대조·개정 추적) | 준수·대외 절차 **최우선 축**, **문서·보고** 중심. 반복 독해·개정 추적·체크리스트 **부담**   | 주제별 요약·목차 초안, 개정 요지 후보(**원문·공지 대조**), 보고·심의·계약 단계별 **질문–증빙 후보–채울 사실** 표 초안 |
| **3** | 감사·점검 문서·매핑 (통제 ↔ AWS·절차·증빙)               | 내·외부·감독 점검 빈도, 통제별 설정·로그·절차 부착 **문서·증빙** 작업 다수               | 증빙 종류·체크리스트 초안, 운영 절차·아키텍처 **질문형 빈칸**, 문장·표 형식 통일 보조                       |
