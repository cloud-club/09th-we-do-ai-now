# We Do AI Now 스터디 5주차 진행 요약

## 세션 개요

- 일자: 2026-04-12
- 주제: AI 사용 점검과 직무별 Playbook 초안 리뷰
- 참여자: 박준우, 소병욱, 김근식, 우다현, 문영호, 김지원
- 진행 목적: 외부 사례나 트렌드 정리에서 한 걸음 더 나아가, 각자가 실제로 AI를 어디에 쓰고 있고 어디에서 막히는지 점검한 뒤 직무별 첫 AI Playbook 초안을 서로 리뷰하는 것

## 이번 주 핵심 논의

- 이번 회차는 "어떤 AI가 뜨는가"보다 "나는 지금 AI를 어떻게 쓰고 있으며, 무엇을 반복 가능한 방식으로 남기고 있는가"를 점검하는 실사용 회고 성격이 강했다.
- 발표는 프리세일즈 미팅 요약과 제안 템플릿, 클라우드 운영 자동화, DevOps 에이전트 플랫폼, 트러블슈팅/문서화 아카이빙, 하네스와 서브 에이전트, 취업 준비용 맥락 자산화까지 넓게 퍼졌지만, 모두 반복 가능한 입력 구조와 검증 루틴, 지식 축적의 필요성으로 수렴했다.
- 세션 초반의 PR 머지, Open Code/Codex 사용 경험, ClovaNote 다운로드 자동화 이야기도 결국 "좋은 모델을 쓰는 것"보다 "내 흐름 안에 어떻게 연결해 둘 것인가"가 더 중요하다는 방향으로 이어졌다.
- 이번 주차는 저장소에 올라온 개인 과제 초안이 실제 발표와 피드백을 거치며 더 실천적인 형태로 정리된 회차였고, 마지막 세션 전 단계에서 각자의 실행 단위를 더 좁히는 역할을 했다.

## 직무별로 드러난 포인트

- 김근식은 Presales 흐름에서 ClovaNote, GPT, MS Copilot, 메일/제안 템플릿을 조합해 고객 발언을 핵심 니즈, 제안 포인트, 후속 액션으로 빠르게 변환하는 구조를 만들고 싶어 했다.
- 우다현은 웹/익스텐션 중심의 간편한 사용 방식에서 출발했지만, 운영 업무에 실제로 닿으려면 CLI 기반 자동화와 재사용 가능한 템플릿, 검증 루틴이 필요하다고 정리했다.
- 문영호는 DevOps 및 내부 플랫폼 관점에서 테스트 코드, RBAC, AI 샌드박스, 사용자 피드백 루프까지 갖춰져야 에이전트가 운영 친화적으로 굴러갈 수 있다고 설명했다.
- 박준우는 트러블슈팅 로그, 운영 문서, Runbook, STT 세션 정리 자체를 다시 쓰일 수 있는 자산으로 보고, Obsidian과 Codex 기반 아카이빙 구조를 중심으로 활용 사례를 공유했다.
- 소병욱은 파운데이션 모델 자체보다 목표 정의, 정답 상태, 하네스, 서브 에이전트, LLM Wiki 같은 상위 구조가 더 중요하다는 빌더 관점을 강조했다.
- 김지원은 취업 준비 맥락에서 Notion에 쌓인 개인 정보와 JD를 연결하고, 역할-목적-제약-출력 형식을 먼저 고정한 뒤 자소서와 면접 준비 결과를 템플릿 자산으로 남기는 방식을 소개했다.

## 공통 인사이트

- 이번 세션을 관통한 결론은 좋은 모델 하나보다 반복 가능한 입력 구조, 검증, 보관, 재사용이 더 중요하다는 것이었다.
- 참석자들은 프롬프트를 잘 쓰는 것만으로는 한계가 있고, 자료 검색부터 정리, 요약, 검토, 전달, 아카이빙까지 포함한 전체 파이프라인이 설계되어야 AI가 업무 안에 정착할 수 있다고 봤다.
- 단발성으로 잘 나온 결과보다, 잘된 사례와 실패 사례를 함께 축적해 다음 작업의 컨텍스트로 재사용하는 구조가 Playbook의 핵심이라는 인식이 강하게 드러났다.
- 이 주차는 이전 주차에서 찾은 반복 업무 후보를 실제 사용 흐름으로 연결해 본 단계였고, 마지막 오프라인 세션에서는 각자 작은 결과물을 직접 만들어 보는 해커톤형 작업 세션으로 이어질 가능성이 커졌다.

## 다음 주로 이어진 과제

- 각자 5주차 Playbook 초안을 자신의 실제 업무나 학습 맥락에 더 맞게 다듬어 완성본 형태로 정리한다.
- 반복 업무 1개를 골라 CLI, 템플릿, 에이전트, 아카이빙 중 한 가지 방식으로 작게 실험하고 결과를 기록한다.
- 잘 된 프롬프트, 템플릿, 스킬, 컨텍스트 문서를 자신에게 맞는 저장소에 축적한다.
- 마지막 오프라인 세션의 공간, 예산, 해커톤 형식 여부를 확정하고 작업 결과물을 만들 준비를 한다.
