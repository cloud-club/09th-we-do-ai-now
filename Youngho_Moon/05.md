# 현재 나의 AI 사용 방식

1-1. 현재 사용 중인 도구
사용 도구: claude code, gemini cli, codex, GLM, 자체개발 tool
주 사용 목적: 코드 작성 및 테스트, 문서 작성 및 요약, 오픈소스 코드 분석, 회사 자체 tool 제공
사용 방식(Web / Extension / CLI / API 등): CLI, API

1-2. 자주 사용하는 패턴
내가 가장 자주 하는 AI 활용 방식: 코드 분석, 개발, CLI, 사용자 편의도구 개발
이 방식을 자주 쓰는 이유: 시간 단축을 위해서, 반복 작업을 줄이기 위해서
현재 방식의 장점: 규모가 큰 오픈소스여도, 혹은 프로젝트여도 빠른 시간안에 파악이 가능함
현재 방식의 한계: 컨텍스트가 많아지면, 병목이 많아짐

실패 패턴 분석
3-1. 기대만큼 잘 되지 않았던 경험
사례: 사내 플랫폼 개발 시에 많은 토큰 대비 좋지 않은 결과
당시 기대했던 결과: AI를 통한 ManDay 절약
실제 결과: 디버깅이 힘들어, 오히려 시간이 늘어남
왜 그렇게 되었는가?

1. 테스트 코드의 부재
2. spec 명시 및 문서화가 잘 안되어있는 부분
3. 권한 통제의 부재

3-2. 실패 원인 정리
프롬프트 문제: 너무 추상적인 문제
컨텍스트 문제: 권한 부여가 되어있지 않아, 자유롭게 사내 시스템에 접근이 불가능함
검증 문제: 테스트 코드의 부재
도구 선택 문제: 라이브러리나 프레임워크에 대한 무한 신뢰
사용 습관 문제: 너무 추상적으로만 작업하려고 하는것

3-3. 앞으로의 개선 원칙
반복하지 않기 위해 필요한 원칙 3가지:

1. 테스트 코드를 통한 무한 검증
2. 환경에 맞는 도구 선택
3. 마이크로 매니징

내 직무에 맞는 AI 적용 전략
4-1. 내 직무와 핵심 업무
나의 직무: DevOps 엔지니어
핵심 업무: 개발자들의 비 효율적인 부분 제거
AI가 가장 먼저 들어가야 할 업무: 모든 업무

4-2. AI 적용 우선순위

1. 오픈소스 코드 분석
2. 개발자와의 간극 줄이기
3. 사내 시스템 개선

4-3. 나에게 맞는 활용 방식
CLI/API 자동화
Agent/Workflow 확장 가능성:

- 개발자들이 쉽게 사용할 수 있는 플랫폼 제작
- 주간 보고서

4-4. 왜 이 방식이 내 직무에 맞는가
직무 특성과의 연결: 모니터링 에이전트 개발, Q&A 봇
기대 효과: 커무니케이션 비용 감소, 개발자들의 만족도 증가
필요한 전제 조건: 회사가 비용에 대한 인식이 개선되어야함
