# 직무의 가치 정의

❓ 이 직무는 조직, 고객, 서비스에 어떤 가치를 제공하는가?

- 조직에서 겪고 있는 문제를 해결하여, 조직의 생산성을 향상시킨다.
- 고객에게는 조직의 생산성 향상으로 인한, 빠른 개발 사이클 주기로 인해서 더 좋은 제품을 빠르게 전달할 수 있다.
- 서비스에는 안정적인 환경을 제공하여, 사용자의 불편함 없이 서비스가 운영되도록 한다.

❓ 이 직무가 잘 수행되면 무엇이 좋아지고, 부족하면 어떤 문제가 생기는가?

- 잘 수행 되는 경우 개발 사이클이 빨라지면서, 신규 기능 추가가 빠르게 가능하다.
- 부족한 경우 개발 사이클 저하로 인한 신규 기능 추가 부족 및 기능의 안정성이 줄어든다.

# 직무의 성과 기준 정의

❓ 이 직무의 성과를 보여주는 정량 지표에는 어떤 것들이 있는가?

- SLI, SLO
- 비용 절감 수치
- 배포 사이클

❓ 이 직무의 성과를 보여주는 정성 지표에는 어떤 것들이 있는가?

- 개발 사이클 및 개발자 만족도
- 장애 대응 시간

# 직무의 업무 분해

❓ 이 직무의 업무를 주 업무 / 부 업무 / 기타 업무로 나누면 어떻게 구분할 수 있는가?

- 주 업무 : CI/CD 파이프라인 구축, 모니터링 시스템 구축, 클러스터 관리 및 운영, 이외의 공통적인 문제해결
- 부 업무 : 코드 퀄리티 향상
- 기타 업무 : 개발 문화 전파

❓ 여러 업무 중 반복이 많거나, 시간이 많이 들거나, 실수 위험이 큰 업무는 무엇인가?

- 반복은 없음
- 시간이 많이 드는 작업: 공통적인 문제 해결을 위한 POC
- 실수가 큰 업무: 클러스터 작업

# AI 우선 적용 영역 선정

❓ 여러 업무 중 AI를 가장 먼저 적용하면 효과가 클 것 같은 업무 1~2개는 무엇인가?

- 공통적인 문제 해결을 위한 POC
- 클러스터 작업

❓ 그 업무에 AI가 필요하다고 판단한 이유는 무엇이며, 어떤 방식으로 도움을 줄 수 있을까?

- 공통적인 문제 해결을 위한 POC : POC 과정 시에 테스트 환경 구성을 AI 통해 진행
- 클러스터 작업 : 실수를 방지하기 위해, 자동화 스크립트 구축 혹은 스크립트 검증을 진행
