## 실패 패턴 분석
### 3-1. 기대만큼 잘 되지 않았던 경험
- 사례: 바이브 코딩을 기대했으나 커스터마이징이 어려웠음
- 당시 기대했던 결과: 자연어만으로 개발을 거의 자동화
- 실제 결과: 결과물의 완성도와 일관성이 부족함
- 왜 그렇게 되었는가? → 전체 시스템 맥락과 요구사항을 충분히 전달하지 못했기 때문

### 3-2. 실패 원인 정리
- 프롬프트 문제: 요구사항이 구체적이지 않고 단계적으로 나누지 않음
- 컨텍스트 문제: 프로젝트 전체 구조/목표 없이 단편적인 요청만 전달
- 검증 문제: 결과를 그대로 사용하고, 검증/리팩토링 과정이 부족함
- 도구 선택 문제: 상황에 맞는 도구 선택 없이 하나의 도구에 의존
- 사용 습관 문제: “일회성 질문 → 결과 확인” 수준에서 끝남

### 3-3. 앞으로의 개선 원칙
- 반복하지 않기 위해 필요한 원칙 3가지:
1) 작업을 단계 단위로 쪼개서 요청하기
2) 프로젝트 단위로 컨텍스트를 명확히 정리해서 전달하기
3) 결과를 반드시 검증하고 개선하는 루프 만들기

## 내 직무에 맞는 AI 적용 전략
### 4-1. 내 직무와 핵심 업무
- 나의 직무: 클라우드 플랫폼 엔지니어
- 핵심 업무: 클라우드 운영
- AI가 가장 먼저 들어가야 할 업무: 반복 업무 자동화

### 4-2. AI 적용 우선순위
1. 운영 작업 자동화
2. 리포트 생성 자동화
3. 사용자 응답 자동화

### 4-3. 나에게 맞는 활용 방식
- 웹형 대화: 빠른 문제 해결, 개념 정리, 코드 스니펫 생성에 활용
- 비교형 사용: 여러 AI(ChatGPT, Gemini, Claude)를 동시에 활용해 결과 비교 및 보완
- CLI/API 자동화: 스크립트 생성, IaC, 운영 작업을 자동화하는 방향으로 확장
- 문서/지식 아카이빙 연계: 자주 사용하는 프롬프트, 트러블슈팅, 코드 패턴을 문서화하여 재사용
- Agent/Workflow 확장 가능성: 반복 업무를 기반으로 자동 실행되는 워크플로우(배포, 점검 등)로 확장 가능

### 4-4. 왜 이 방식이 내 직무에 맞는가
- 직무 특성과의 연결: 클라우드 운영은 반복 작업, 로그 분석, 장애 대응 등 자동화에 적합한 영역이 많음
- 기대 효과: 작업 시간 단축, 휴먼 에러 감소, 운영 효율성 향상
- 필요한 전제 조건: 표준화된 작업 프로세스 정리, 스크립트 기반 환경 구축, 기본적인 API/CLI 활용 능력

## 나의 AI Playbook 초안
### 6-1. 나의 AI 활용 원칙
1. 명확한 요구사항과 맥락을 함께 전달한다
2. 결과를 그대로 사용하지 않고 반드시 검증한다
3. 반복 가능한 형태(템플릿/스크립트)로 축적한다

### 6-2. 도구별 역할 분담
- GPT: 아이디어 확장, 문장 정리, 요약 및 구조화
- Gemini: 정보 탐색 및 빠른 비교, 보조 검증 용도
- Claude: 요약 정리, 프로그래밍
- MSCopilot: PPT 제작, 코드 검색 등에 활용
- Perplexity: 웹서핑
- CLI/API: 반복 작업 자동화, 스크립트 실행, 인프라 제어

### 6-3. 표준 사용 흐름
- 업무 입력
→ AI 활용 (초안 생성 / 코드 생성 / 아이디어 도출)
→ 결과 정리 (필요한 부분만 선별 및 구조화)
→ 검증 (기술적/논리적 검토)
→ 최종 산출물화 (문서, 코드, 자동화 스크립트 등으로 정리)

### 6-4. 검증 루틴
- 사실 검증: 공식 문서, 레퍼런스 자료와 비교하여 정확성 확인
- 기술 검증: 코드 실행, 테스트 환경에서 동작 여부 확인
- 문서 검증: 문맥 흐름, 오타, 전달력 점검
- 실행 가능성 검증: 실제 업무 환경에서 적용 가능한지 확인 (권한, 환경, 리스크 등)

### 6-5. 반복 사용할 템플릿
- 템플릿 1: 운영 작업 자동화 요청 템플릿
→ “현재 작업 내용 / 목표 / 환경 / 제약 조건 / 원하는 결과 형태”를 포함하여 스크립트 생성 요청

- 템플릿 2: 트러블슈팅 분석 템플릿
→ “문제 상황 / 발생 환경 / 로그 / 시도한 해결 방법 / 원하는 결과”를 기반으로 원인 분석 요청

- 템플릿 3: 문서/보고서 생성 템플릿
→ “목적 / 대상 / 포함할 내용 / 형식(보고서, 요약 등)”을 명확히 전달하여 결과 생성

### 6-6. 다음 2주 실행 계획
- 바로 적용할 업무: 반복적으로 수행하는 운영 작업 1개 선정 (예: 배포, 로그 확인 등)

- 실험할 방식: AI로 스크립트 초안 생성 → 수정 → 실제 적용 → 결과 기록 및 템플릿화

- 기대 효과: 작업 시간 단축, 반복 작업 감소, 자동화 기반 업무 전환

- 측정 기준:  작업 수행 시간 변화
→ 수작업 대비 자동화 비율
→ 재사용 가능한 스크립트/템플릿 개수
