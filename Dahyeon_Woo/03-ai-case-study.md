## 1. 사례 기본 정보

- **선정 사례**: 카카오페이 DevOps팀의 AI 업무 도우미 ‘춘시리(Choonsiri)’  
- **링크**: https://tech.kakaopay.com/post/choonsiri/
- **선정 이유**: 
    일반적인 업무 환경, 특히 현재의 업무 환경에서도 적용 가능성이 높다는 점에서 해당 사례를 선정하였다.

    특히 티켓 정리, 업무 요약, 보고서 작성과 같은 반복적인 업무를 자동화하면서도, 팀 전체의 협업 방식까지 변화시켰다는 점이 인상적이었다.  
    
    아울러 클라우드 및 운영 직무에 직접적으로 적용 가능한 구조를 가지고 있어 본 사례를 선택하게 되었다.


## 2. 문제 정의

### 핵심 문제
  DevOps 환경에서 발생하는 다양한 업무 티켓과 작업 기록이 분산되어 있어 이를 정리하고 공유하는 데 많은 시간이 소요되는 문제

### 기존 방식의 한계
  - 팀원들이 각자의 티켓을 수동으로 정리해야 함
  - 보고를 위해 이를 다시 취합하고 요약하는 이중 작업 발생
  - 업무 기록은 많지만 핵심 내용만 빠르게 파악하기 어려움
  - 반복적인 보고서 작성 업무로 인해 생산성이 저하됨


## 3. 해결 방식

### AI 적용 지점
  - 문서/보고서 작성
  - 티켓 수집 → 분석 → 요약 → 보고서 생성 전 과정
  - 즉, “정보 정리 및 해석 단계” 전체에 AI 적용

### 구현 방식

1. 업무 티켓 및 작업 데이터 수집  
2. AI가 데이터를 분석하고 요약  
3. 팀 단위 업무 내용을 자동 정리  
4. 사용자는 결과를 확인하고 활용  

### 업무 방식 변화

#### Before
- 팀원이 티켓 정리  
- 보고를 위한 요약  
- 보고서 수동 작성  

#### After
- AI가 티켓 자동 수집 및 요약  
- 업무 흐름 자동 정리  
- 클릭 한 번으로 보고서 생성  


## 4. 성과

### 핵심 성과
  - 주간 보고 작성 시간 단축
  - 반복 업무 감소
  - 팀 전체 업무 가시성 향상
  - 협업 효율 증가

### 성과가 발생한 이유

기존에는 사람이  
→ 정보를 모으고 → 정리하고 → 요약하는 과정을 수행했다.

하지만 춘시리 도입 이후에는  
→ AI가 정보 수집, 정리, 요약을 자동으로 수행한다.

> 단순 자동화를 넘어서, **정보 해석 과정이 AI로 이전되었다.**


## 5. 조건 / 리스크

### 필요 조건
  - 데이터 접근 및 통합 환경 구축
  - AI가 이해할 수 있는 데이터 구조 필요

### 리스크
  - AI의 잘못된 요약 가능성 (Hallucination)
  - 민감 정보 노출 위험
  - 결과에 대한 과신으로 검수 부족 발생 가능

> 따라서  **AI 결과를 검증하는 프로세스가 반드시 필요하다**


## 6. 내 업무 적용

### 핵심 원리
  반복적이고 시간 소모적인 ‘정보 정리 및 요약 작업’을 AI에 위임한다

### 적용 아이디어

1. 운영 기록을 자동으로 문서화  
2. 티켓 기반 업무 내용을 자동으로 보고서 형태로 변환  
3. 로그 데이터를 AI로 요약하여 장애 상황 자동 정리  


## 7. 느낀 점 / 내 생각

생각보다 AI를 실제 업무에 적용하여 성과를 만들고 이를 구체적으로 공개한 사례는 많지 않았다.  
그만큼 AI를 도입하는 것보다, 이를 업무 흐름에 맞게 적용하는 것이 더 어려운 문제라는 생각이 들었다.

이번 사례를 통해 업무에서 AI를 어디에 활용해야 하는지에 대해 다시 고민해보게 되었다.  
모든 업무에 무조건 AI를 적용하기보다는, 필요할 때 선택적으로 활용하는 것이 중요하다고 느꼈다.

특히 시간이 많이 소요되는 문제 이해나 정보 정리 과정과 같이, 반복적이면서도 인지 부담이 큰 작업에는 AI를 적극적으로 활용해도 좋을 것 같다.  
앞으로는 이러한 기준을 바탕으로 AI를 도입할 지점을 판단하고, 업무 효율화를 시도해보고 싶다.
