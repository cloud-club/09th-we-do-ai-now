# 1. 현재 나의 AI 사용 방식 점검

## Pre-sales 업무에서의 활용
AI를 개발 도구가 아닌 **고객 대응 / 제안 준비 보조 도구**로 사용

### 1) 고객 정보 구조화
- 고객 요구사항, 제품 기능, 옵션, 조건 등을 빠르게 구조화할 때 활용
- 예시:
  - 고객사 미팅 녹음본 → 텍스트 추출 → AI로 핵심 정보 정리
  - 제품 기능 / 옵션 / 용어 / 조건 등 흩어진 정보를 정리하여 빠르게 미팅 내용 파악

### 2) 문서 작성 보조
- 메일, 제안 문구, 소개 자료, 문의 답변 등 문장 정리에 활용
- 예시:
  - 모든 문서 작업 시 AI를 통해 맞춤법, 띄어쓰기 검토 수행

---

# 2. 지금까지의 사용에서 아쉬웠던 점

- 필요할 때마다 사용하는 방식 → **반복 가능한 템플릿으로 축적되지 않음**
- 결과물이 좋아 보여도 **업무 흐름에 완전히 내재화되지 않음**
  - 따라서 결과물은 항상 검토 후 제출 필요

---

# 3. 실패 패턴 분석

- 질문이 너무 넓고 추상적일 경우 → **일반적인 답변만 생성됨**
- 기능 설명 중심 접근 시 → **고객 관점 메시지가 약해짐**
- 가장 필요한 기능:
  - 고객사 이전 사례
  - RFP
  - 공수 산정
  - 번들 견적서

→ 현재 사내 AI 도구에서 지원되지 않아  
→ **회사 공유 폴더에서 직접 탐색 중**

---

# 4. 내 직무에 맞는 AI 적용 전략
- 고객 발언을 정리 → **핵심 니즈 + 제안 포인트로 변환**
- 동일한 내용도 대상에 맞게 변환:
  - 기술지원팀
  - SA
  - 사용자
  - SR(영업)
- 반복 문서 업무 자동화:
  - 미팅 요약
  - 고객 질문 정리
  - 제안 초안
  - 메일 초안

### 핵심 목표
> 특정 고객사 관련 파일/폴더를 AI로 즉시 검색하여  
> 필요한 자료에 빠르게 접근할 수 있도록 하는 것

---

# 5. 앞으로 해보고 싶은 AI 활용 방향

- 고객 문의를 **유형별로 분류하여 답변 템플릿화**
- 잘된 결과뿐 아니라:
  - 아쉬웠던 사례
  - 실패 이유  
  → 함께 기록하여 개선
- 영업사원 / 엔지니어 / SA의 **정확한 스케줄 관리 및 반영**
