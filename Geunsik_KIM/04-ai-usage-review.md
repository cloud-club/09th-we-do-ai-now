# 1. 현재 나의 AI 사용 방식 점검

## Pre-sales 업무에서의 활용
AI를 개발 도구가 아닌 **고객 대응 / 제안 준비 보조 도구**로 사용

### 1) 고객 정보 구조화
- 고객 요구사항, 제품 기능, 옵션, 조건 등을 빠르게 구조화할 때 활용
- 예시:
  - 고객사 미팅 녹음본 → 텍스트 추출 → AI로 핵심 정보 정리
  - 제품 기능 / 옵션 / 용어 / 조건 등 흩어진 정보를 정리하여 빠르게 미팅 내용 파악

### 2) 문서 작성 보조
- 메일, 제안 문구, 소개 자료, 문의 답변 등 문장 정리에 활용
- 예시:
  - 모든 문서 작업 시 AI를 통해 맞춤법, 띄어쓰기 검토 수행

---

# 2. 지금까지의 사용에서 아쉬웠던 점

- 필요할 때마다 사용하는 방식 → **반복 가능한 템플릿으로 축적되지 않음**
- 결과물이 좋아 보여도 **업무 흐름에 완전히 내재화되지 않음**
  - 따라서 결과물은 항상 검토 후 제출 필요

---

# 3. 실패 패턴 분석

- 질문이 너무 넓고 추상적일 경우 → **일반적인 답변만 생성됨**
- 기능 설명 중심 접근 시 → **고객 관점 메시지가 약해짐**
- 가장 필요한 기능:
  - 고객사 이전 사례
  - RFP
  - 공수 산정
  - 번들 견적서

→ 현재 사내 AI 도구에서 지원되지 않아  
→ **회사 공유 폴더에서 직접 탐색 중**

---

# 4. 내 직무에 맞는 AI 적용 전략
- 고객 발언을 정리 → **핵심 니즈 + 제안 포인트로 변환**
- 동일한 내용도 대상에 맞게 변환:
  - 기술지원팀
  - SA
  - 사용자
  - SR(영업)
- 반복 문서 업무 자동화:
  - 미팅 요약
  - 고객 질문 정리
  - 제안 초안
  - 메일 초안

### 핵심 목표
> 특정 고객사 관련 파일/폴더를 AI로 즉시 검색하여  
> 필요한 자료에 빠르게 접근할 수 있도록 하는 것

---

# 5. 앞으로 해보고 싶은 AI 활용 방향

- 고객 문의를 **유형별로 분류하여 답변 템플릿화**
- 잘된 결과뿐 아니라:
  - 아쉬웠던 사례
  - 실패 이유  
  → 함께 기록하여 개선
- 영업사원 / 엔지니어 / SA의 **정확한 스케줄 관리 및 반영**

# 6. PlayBook 초안
## 6-1. 나의 AI 활용 원칙

1. 추상적인 질문이 아닌, 상세하고 구체적으로 질문을 하자.
2. 각 상황에 맞는 AI 툴을 사용하자.
3. 필요하다면 프롬프트 작성에만 집중하지 말고, 자료 검색·정리·요약·검토 흐름까지 고려해 AI가 업무 안에서 제대로 작동할 수 있도록 활용하자.

---

## 6-2. 도구별 역할 분담

- **GPT**: 미팅 내용 요약, 고객 요구사항 정리, 제안 문구 초안 작성, 메일 문장 다듬기, 고객 상황에 맞는 설명 방식 변환에 활용
- **Gemini Nano Banana**: 제안서나 교육자료를 만들 때, 상황에 맞는 이미지나 시각 자료가 필요할 경우 생성하는 용도로 활용
- **MSCopilot**: Outlook, Teams, Word, PowerPoint 등 M365 환경 안에서 회의 내용 정리, 문서 초안 작성 등 사내 자료 기반 업무 보조에 활용
- **Clova Note**: 고객사 미팅 녹음본을 텍스트로 추출하고, 대화 중 흩어져 나온 제품 기능, 옵션, 용어, 조건 등의 핵심 내용을 정리하는 데 활용

---

## 6-3. 표준 사용 흐름

1. **업무 목적과 대상부터 먼저 정리**
   - 누구를 위한 결과물인지 확인
   - 고객 설명용인지, 내부 공유용인지, 제안서용인지 먼저 구분

2. **필요한 자료를 먼저 모으기**
   - 미팅 녹음본, 회의록, 기존 제안서, 고객 요청사항, 관련 문서 확인
   - 필요한 경우 텍스트 추출 후 정리 가능한 상태로 준비

3. **AI에 넣기 전에 입력 정보를 구조화**
   - 고객명
   - 고객 니즈
   - 제품 기능/옵션
   - 조건 및 일정
   - 요청 목적

4. **상황에 맞는 AI 도구 선택**
   - 회의 내용 정리: Clova Note + GPT
   - 문장 다듬기/초안 작성: GPT
   - 이미지 생성: Gemini Banana
   - M365 문서 흐름 연계: MS Copilot

5. **결과물 초안 생성**
   - 요약
   - 고객 니즈 정리
   - 제안 포인트 정리

6. **사람이 최종 검토 후 수정**
   - 표현이 어색하지 않은지, 너무 일반적이지 않은지 확인
   - 실제 고객 상황과 맞는지 확인

---


## 6-4. 반복 사용할 템플릿

### 템플릿 1: 미팅 요약 템플릿
- 고객명
- 미팅 목적
- 핵심 요구사항
- 언급된 제품 기능/옵션
- 조건 및 일정
- 후속 액션

### 템플릿 2: 고객 맞춤 제안 정리 템플릿
- 고객 업종/상황
- 현재 문제
- 제안 포인트
- 추천 기능
- 기대 효과
- 예상 질문

### 템플릿 3: 메일/공유 문서 초안 템플릿
- 전달 목적
- 핵심 내용 요약
- 요청사항 또는 안내사항
- 후속 일정
- 마무리 문구

---

## 6-5. 다음 2주 실행 계획

- **바로 적용할 업무**:  
  고객사 미팅 녹음본 요약, 미팅 후 공유 메일 초안 작성, 제안 포인트 정리 업무에 우선 적용

- **실험할 방식**:  
  회의 내용을 바로 요약해달라고 하기보다, 고객 요구사항 / 제품 기능 / 조건 / 일정 / 후속 액션으로 나누어 정리하도록 구조화해서 요청  
  같은 내용도 내부 공유용, 고객 설명용으로 나눠서 결과 차이 비교

- **기대 효과**:  
  미팅 후 정리 시간 단축  
  문서 초안 작성 속도 향상  
  고객 맞춤 설명의 일관성 확보  
  반복 업무 부담 감소

- **측정 기준**:  
  미팅 후 정리 소요 시간  
  메일/초안 작성 시간
