# Domain Specific-Language Model

### 도메인 특화 언어 모델이란?
범용 LLM을 회사 내부 데이터로 Fine-tuning 시켜 회사 전용 모델을 만드는 방식
- Fine Tuning: 방대한 데이터를 학습시킨 사전 학습 모델을 가져와서, 특정 작업이나 다양한 도메인에 맞춰 추가 학습을 시키는 과정을 의미한다.
  - 작동 원리
      1. 기존 지식 보존: 모델의 앞부분(입력에 가까운 층)은 언어의 기초적인 특징을 담고 있기 때문에 보통 그대로 둔다.
      2. 출력층 수정: 모델의 마지막 부분(출력층)을 우리가 원하는 목적에 맞게 교체한다.
      3. 추가 학습: 준비한 특정 데이터를 입력하여 모델의 가중치를 업데이트 시킨다.
   
  - 주요 파인튜닝 기법
      1. Full Fine Tuning: 모델의 모든 가중치를 다시 업데이트하는 방식. 성능은 좋지만 자원이 많이 든다.
      2. LoRA (Low-Rank Adaptation): 전체를 건드리지 않고, 일부 핵심적인 '어댑터'만 추가하여 학습시키는 방식.
      3. Instruction Fine-tuning: 지시어(Instruction)와 정답 쌍을 학습시켜 대화 능력을 높이는 방식. (ex. 요약해줘, 변역해줘 와 같은 지시어)

## LoRA(Low-Rank Adaption)
기존의 파인 튜닝은 이 행렬의 모든 숫자를 직접 수정해야 하므로, 상당한 컴퓨팅 파워와 메모리가 필요하기 때문에, 상황에 따라 LoRA를 통해 LoRA를 통해 수조 개의 파라미터를 직접 수정하는 대신, 아주 적은 수의 어댑터 파라미터만을 학습시켜 효율성을 높일 수 있다.
1. Freeze & Add: LoRA는 기존 모델의 지식인 사전 학습된 가중치를 건들지 않고 절대 수정하지 않는다. 대신 그 옆에 Adapter라고 불리는 아주 얇은 행어로 된 보조 장치를 추가하여 변화량만 학습시킨다.
2. 행렬 분해 (Matrix Decomposition): LoRA의 핵심은 거대한 행렬을 수정하는 것이 아닌, 두 개의 작은 행렬로 쪼개는 것이다. (ex. 원래 수정해야 할 행렬이 100*100(10,000개 요소), 행렬 A: 100*8 (800개), 행렬 B: 8*100 (800개)로 분해한다. 이는 결과적으로 1,600개의 숫자만 학습하면 된다. 
3. Rank-> 학습의 통로: 위에서 숫자 8과 같은 작은 값을 Rank라고 한다.이는 새로운 지식을 전달하는 통로의 폭과 같다. Rank가 작을수록 학습 속도가 빠르고 메모리를 적게 쓰며, Rank가 클수록 더 복잡하고 세밀한 지식을 학습할 수 있습니다.
4. 병합: 실제 추론(Inference) 시에는 기존 지식+행렬(A*B)를 더해서 계산한다. 이를 통해 모델의 원래 성능은 유지하면서도, 우리가 원하는 목적에 맞춰 출력을 생성할 수 있다.



## Instruction Fine-tuning 
사전 학습(Pre-training)만 마친 모델은 문장을 이어 쓰는 데는 능숙하지만, "이 글을 요약해줘" 같은 구체적인 명령에는 관련 내용을 나열하기만 할 뿐 실제 요약 작업을 수행하지 못하는 경우가 많다. 이를 해결하기 위해 지시어와 정답 쌍을 학습시킨다.

1. 지시어-응답 쌍 데이터셋: 단순히 텍스트를 읽히는 것이 아니라, 지시(Instruction), 입력(Input), 출력(Output)의 3단 구조로 된 데이터를 학습시킨다. (ex. 지시: "번역해줘", 입력: "Hello", 출력: "안녕하세요")
2. 대화형 인터페이스 최적화: 모델이 사용자의 의도(Intent)를 파악하도록 훈련한다. 이 과정을 통해 모델은 "말을 알아듣는 챗봇"의 형태를 갖추게 된다.
3. 제로샷(Zero-shot) 능력 향상: 한 번도 본 적 없는 유형의 지시가 내려져도, 학습된 지시어 패턴을 바탕으로 응답할 수 있는 일반화 능력이 생긴다. 이는 모델이 지시어의 '문맥'과 '의도'를 파악하는 법을 배웠기 때문이다.

### ❓ 왜 이 키워드를 선택했는가?
현재 회사에선 범용 LLM 기반의 AI 도구인 'AI-MAX' 프로젝트를 추진 중이다.
기업 고객이 사내 폐쇄망에 보관된 내부 데이터를 안전하게 조회, 분석하고 업무에 활용할 수 있도록, 맞춤형 컨설팅을 통한 최적의 AI 업무 환경 구축을 목표로 하고 있다.
지금은 RAG를 활용해 각 기업의 업무 데이터를 실시간으로 참조하는 구조로 진행되고 있지만, 보다 정확하고 정교한 출력값을 얻기 위해서는 Domain-Specific LM을 통해 AI-MAX의 문맥 이해도와 답변 전문성을 한층 더 내재화시킬 수 있다고 판단하여 해당 키워드를 선택했습니다.

저는 해당 AI-MAX팀에 소속되어 있진 않습니다.

현재 회사 내부에서도 AI-MAX를 추진 중이다. 단순 RAG 구조에서 파인 튜닝된 DSLM으로 고도화할 경우, 회사만의 고유한 업무 프로세스와 인적 자원 정보를 모델이 직접 체득하게 된다. 결과적으로 영업과 기술 지원 부서 간의 복잡한 협업 스케줄 최적화나 고객사별 맞춤형 영업 전략 수립에 있어 훨씬 높은 정확도와 신뢰성을 확보할 수 있다고 생각합니다.
