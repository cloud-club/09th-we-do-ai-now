## 2. 이직 준비용

### 1.1 나의 AI 활용 원칙

1. **막연하게 질문하지 않고, 목적을 먼저 정리한다.**  
   AI에게 “자기소개서 써줘”, “면접 준비해줘”라고 바로 요청하지 않고, 지원 기업, 직무, 채용공고, 내 경험, 원하는 답변 방향을 함께 전달한다.

2. **이직 준비 단계별로 AI 역할을 나눈다.**  
   채용공고 분석, 경험 정리, 자기소개서 작성, 면접 예상 질문 생성, 답변 검토 등 단계마다 AI를 다르게 활용한다.

3. **AI가 만든 문장을 그대로 사용하지 않는다.**  
   AI 결과물은 초안으로만 사용하고, 최종 문장은 내 말투와 실제 경험에 맞게 수정한다.

4. **내 경험을 먼저 정리한 뒤 AI를 사용한다.**  
   경험이 부족하게 입력되면 AI 결과도 일반적으로 나온다. 따라서 프로젝트, 기술 스택, 문제 해결 경험, 성과를 먼저 정리한 뒤 활용한다.

5. **지원 직무와 연결되는 결과물을 만드는 데 집중한다.**  
   단순히 보기 좋은 자기소개서보다, 채용공고에서 요구하는 역량과 내 경험이 자연스럽게 연결되는지 확인한다.

### 1.2 도구별 역할 분담

#### ChatGPT / 사내 AI 도구

1. 채용공고 분석에 활용한다.
2. 직무에서 요구하는 핵심 역량을 추출한다.
3. 내 경험과 요구 역량을 연결한다.
4. 자기소개서 문항별 소재를 정리한다.
5. 면접 예상 질문을 생성한다.
6. 답변 초안을 만들고 문장을 다듬는다.
7. 너무 AI가 쓴 것처럼 보이는 표현을 자연스럽게 수정한다.

#### Notion / 문서 정리 도구

1. 지원 기업별 자료를 정리한다.
2. 채용공고 원문을 저장한다.
3. 기업 분석 내용, 자기소개서 초안, 면접 질문을 한곳에 모은다.
4. 내 프로젝트 경험과 기술 스택을 정리한다.
5. 지원 현황을 관리한다.

예시:

```text
기업명:
지원 직무:
마감일:
채용공고 링크:
핵심 요구 역량:
연결 가능한 내 경험:
자기소개서 문항:
면접 예상 질문:
지원 상태:
```

#### GitHub

1. 프로젝트 코드를 정리한다.
2. README를 취업용으로 개선한다.
3. 프로젝트의 문제 정의, 기술 스택, 아키텍처, 트러블슈팅 경험을 정리한다.
4. 면접에서 설명할 수 있도록 프로젝트별 핵심 포인트를 남긴다.
5. 단순 코드 저장소가 아니라, 내가 어떤 문제를 어떻게 해결했는지 보여주는 자료로 활용한다.

#### Google Drive / OneDrive

1. 이력서, 자기소개서, 포트폴리오 파일을 버전별로 관리한다.
2. 기업별 제출 서류를 따로 보관한다.
3. 면접 답변 자료와 예상 질문 리스트를 저장한다.
4. 필요할 때 AI에 넣을 수 있도록 자료를 정리해둔다.

## 2. 표준 사용 흐름

### Step 1. 지원 목적과 직무를 먼저 정리한다

AI를 사용하기 전에 먼저 아래 내용을 정리한다.

```text
1. 어떤 기업에 지원하는가?
2. 어떤 직무에 지원하는가?
3. 이 직무에서 가장 중요해 보이는 역량은 무엇인가?
4. 내 경험 중 어떤 부분을 강조할 것인가?
5. 이직 사유를 어떻게 설명할 것인가?
```

---

### Step 2. 채용공고를 분석한다

채용공고를 AI에게 넣고, 기업이 원하는 역량을 먼저 뽑는다.

#### 사용 프롬프트

```text
아래 채용공고를 분석해줘.

분석 기준:
1. 주요 업무
2. 필수 역량
3. 우대 역량
4. 기술 키워드
5. 인성/태도 키워드
6. 이 직무에서 가장 중요해 보이는 역량 3가지

채용공고:
[채용공고 붙여넣기]
```

#### 출력 예시

```text
주요 업무:
- 클라우드 인프라 운영
- 서버 및 네트워크 장애 대응
- 배포 자동화
- 모니터링 체계 관리

필수 역량:
- Linux 기본 지식
- 네트워크 이해
- 클라우드 사용 경험
- 문제 해결 능력

우대 역량:
- Docker / Kubernetes 경험
- AWS 경험
- 자동화 스크립트 작성 경험

핵심 역량 3가지:
1. 인프라 기본기
2. 장애 대응 능력
3. 자동화 및 개선 경험
```

---

### Step 3. 내 경험을 정리한다

AI에게 바로 자기소개서를 요청하기 전에, 내 경험을 먼저 정리한다.

```text
1. 백엔드 개발 준비 경험
2. 약 6개월 전부터 인프라 분야로 전환한 경험
3. Linux, Docker, Kubernetes, AWS 학습 경험
4. 프로젝트 배포 및 운영 경험
5. 장애 대응 및 트러블슈팅 경험
6. 2011년부터 2016년까지 중국 국제학교를 다닌 경험
7. 다양한 환경에서 적응하고 소통한 경험
```

#### 경험 정리 프롬프트

```text
아래 내 경험을 이직 준비용으로 정리해줘.

조건:
1. 인프라/클라우드 직무와 연결
2. 백엔드에서 인프라로 전환한 흐름 포함
3. 2011년부터 2016년까지 중국 국제학교를 다닌 경험 포함
4. 너무 과장하지 말고 자연스럽게 정리
5. 자기소개서와 면접에서 모두 활용 가능하게 정리

내 경험:
[내 경험 입력]
```

### Step 4. 채용공고와 내 경험을 매칭한다

채용공고에서 뽑은 요구 역량과 내 경험을 연결한다.

#### 사용 프롬프트

```text
아래 채용공고 분석 결과와 내 경험을 연결해줘.

출력 형식:
1. 요구 역량
2. 연결 가능한 내 경험
3. 자기소개서에서 쓸 수 있는 문장
4. 면접에서 말할 수 있는 답변 방향

채용공고 분석 결과:
[분석 결과 붙여넣기]

내 경험:
[내 경험 붙여넣기]
```

#### 예시

```text
요구 역량:
Linux 서버 운영 이해

연결 가능한 내 경험:
Linux 명령어, systemd, journalctl, 네트워크 설정, 메모리 점검 실습 경험

자기소개서 문장:
Linux 환경에서 서비스 상태와 로그를 확인하며 장애 원인을 추적하는 실습을 반복했습니다.

면접 답변 방향:
실제 장애 상황에서 systemctl, journalctl, ss, free, vmstat 등을 어떤 순서로 확인하는지 중심으로 설명한다.
```

### Step 5. 자기소개서 초안을 작성한다

AI에게 자기소개서 초안을 요청할 때는 문항, 기업, 직무, 내 경험을 함께 넣는다.

#### 사용 프롬프트

```text
아래 정보를 바탕으로 자기소개서 초안을 작성해줘.

조건:
1. 너무 AI가 쓴 것처럼 보이지 않게 작성
2. 과장된 표현은 줄이기
3. 인프라/클라우드 직무와 연결
4. 백엔드에서 인프라로 전환한 흐름 포함
5. 중국 국제학교 경험도 자연스럽게 포함
6. 문장은 담백하게 작성

기업명:
[기업명]

직무:
[직무명]

자기소개서 문항:
[문항]

내 경험:
[내 경험 입력]
```

### Step 6. 자기소개서를 검토한다

초안을 받은 뒤에는 바로 제출하지 않고, 아래 기준으로 다시 검토한다.

#### 검토 프롬프트

```text
아래 자기소개서를 검토해줘.

검토 기준:
1. 직무와 연결이 잘 되는지
2. 문장이 너무 AI처럼 보이지 않는지
3. 경험이 구체적인지
4. 불필요하게 과장된 표현이 있는지
5. 이직 사유가 자연스러운지
6. 면접에서 질문받을 만한 부분은 무엇인지

자기소개서:
[자기소개서 붙여넣기]
```

#### 검토 기준

```text
1. 채용공고의 요구 역량이 반영되었는가?
2. 내 경험이 구체적으로 드러나는가?
3. 백엔드에서 인프라로 전환한 이유가 자연스러운가?
4. 짧은 준비 기간을 성장 가능성으로 설명하고 있는가?
5. 중국 국제학교 경험이 억지스럽지 않게 연결되었는가?
6. 너무 멋있는 표현만 있고 실제 내용이 부족하지 않은가?
```

```text

질문 유형:
1. 인성 질문
2. 이직 사유 질문
3. 직무 전환 질문
4. 기술 질문
5. 프로젝트 질문
6. 꼬리 질문
7. 약점 검증 질문

자기소개서:
[자기소개서]

프로젝트 경험:
[프로젝트 경험]
```

#### 예상 질문 예시

```text
1. 왜 백엔드에서 인프라로 방향을 전환했나요?
```

### Step 8. 면접 답변을 만든다

예상 질문을 받은 뒤, 질문별로 1분 답변을 만든다.

#### 사용 프롬프트

```text
아래 질문에 대한 면접 답변을 작성해줘.

조건:
1. 1분 이내 답변
2. 너무 외운 티가 나지 않게 작성
3. 내 경험 중심으로 작성
4. 인프라/클라우드 직무와 연결
5. 마지막에는 앞으로의 성장 방향 포함
6. 너무 과장하지 말고 자연스럽게 작성

질문:
[질문 입력]

내 배경:
[내 배경 입력]
```

#### 답변 예시

```text
처음에는 백엔드 개발을 중심으로 준비했습니다. API와 서버 개발을 공부하면서 애플리케이션이 어떻게 동작하는지 이해할 수 있었습니다.

그런데 프로젝트를 진행하면서 서비스가 안정적으로 운영되기 위해서는 코드뿐만 아니라 서버 환경, 네트워크, 배포 구조, 모니터링이 함께 중요하다는 것을 느꼈습니다.

그 이후 약 6개월 전부터 인프라와 클라우드 분야를 집중적으로 준비하기 시작했습니다. Linux, Docker, Kubernetes, AWS를 공부하면서 서비스가 실제로 운영되는 기반을 이해하려고 노력하고 있습니다.

저는 백엔드 경험을 버린 것이 아니라, 애플리케이션 구조를 이해하는 인프라 엔지니어로 성장하기 위한 기반으로 활용하고 싶습니다.
```

### Step 9. 꼬리질문을 대비한다

#### 사용 프롬프트

```text

출력 형식:
1. 꼬리질문
2. 질문 의도
3. 답변 방향
4. 주의할 점

답변:
[내 답변 붙여넣기]
```
