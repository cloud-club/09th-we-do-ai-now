# AI Playbook

**작성일: 2026년 4월 26일 | we-do-ai-now 스터디 최종 산출물**

---

## 0. 이 Playbook을 쓰는 이유

> "AI를 쓴다"는 것과 "AI를 잘 쓴다"는 것은 다르다.

지난 몇 달간 AI를 취업 준비를 용도로 매일 썼다.  
면접 녹음을 분석하고, 답변을 구조화하고, 기술 과제 아키텍처를 설계하고, 회사마다 다른 자기소개서를 썼다.

그런데 잘 된 프롬프트도, 검증된 답변 구조도, 나만의 루틴이 없었다는 생각이 들었고, 이 Playbook은 그 과정을 돌아보고 앞으로 어떻게 쓸지를 정리한 문서다.

---

## 1. 내가 AI를 어떻게 써왔나

### 1-1. 실제로 쓴 방식들

**이력서·자기소개서**

- 다양한 회사들의 JD에 맞게 목표 문구와 강점 프레이밍을 조정
- JD를 통째로 붙여넣고 "이 직무에 맞게 내 경험을 어떻게 강조할까"를 물어보는 방식

**인터뷰 준비**

- 면접 녹음 파일을 업로드해서 잘한 점/아쉬운 점 분석
- 2~4차 면접관 스타일별로 전략과 예상 질문·답변 초안 준비
- 면접 직전 "이 질문이 나오면 어떻게 답하지?"를 실시간으로 정리

**기술 과제**

- 클라우드 아키텍처 과제: 계정/네트워크 구조 설계, 컴퓨트/HA, DB/DR 설계를 단계별로 같이 고민
- 설계 다이어그램 구성, PPT 구조, 기술 근거 문서화까지 반복 수정

**회사·JD 리서치**

- 취업 일정 관리 (어느 회사가 언제 바빠질지 예측)
- 경쟁사 포지셔닝 비교, 직무별 핵심 역량 분석

### 1-2. 나의 전형적인 패턴

```
장문의 컨텍스트를 통째로 붙여넣는다
→ 구조화된 결과물을 한 번에 요청한다
→ 초안을 받아서 수동으로 수정한다
→ 대화를 닫는다
→ 다음에 또 처음부터 시작한다
```

빠른 초안 확보에는 효과적이었다.  
하지만 매번 처음부터 시작하는 비용이 컸고, 좋은 결과가 나와도 어디에도 남지 않았다.

## 2. 어디서 실패했나

### 2-1. 가장 기억에 남는 실패

**잘 된 프롬프트를 저장하지 않았다.**

면접 준비할 때 면접관 스타일별로 전략을 짜는 프롬프트가 정말 잘 작동했다.  
그런데 저장하지 않아서 다음 회사 면접에서 비슷한 상황이 왔을 때 처음부터 다시 만들어야 했다.
같은 실수를 반복하면서 "이거 전에도 했던 것 같은데"라는 느낌만 남았다.

### 2-2. 실패 패턴 정리

| 실패 유형            | 구체적인 상황                         | 왜 그랬나                                   |
| -------------------- | ------------------------------------- | ------------------------------------------- |
| **자산화 실패**      | 잘 된 프롬프트·답변 구조를 저장 안 함 | "이번만 쓰고 말지"라는 생각으로 접근        |
| **컨텍스트 소실**    | 며칠에 걸친 작업에서 앞 맥락이 끊김   | 대화를 이어가지 않고 매번 새 창을 열었음    |
| **검증 생략**        | 기술 과제에서 AI 출력을 그대로 신뢰   | 결과가 그럴듯해 보이면 확인을 건너뜀        |
| **도구 선택 무계획** | 모든 작업을 Claude 하나로 처리하려 함 | 도구별 강점을 생각하지 않고 습관적으로 접속 |

### 2-3. 핵심 원인 하나

> AI를 "이번 과제", "이번 면접" 용도로 단발성으로 접근했다.

## 3. 앞으로 어떻게 쓸 것인가

### 3-1. 3가지 원칙

**원칙 1 — 제약부터 명시한다**  
Role, Goal, Constraint, Format을 항상 앞에 쓴다.  
"이력서 고쳐줘"가 아니라 "너는 B2B 기술 직무 채용 전문가야. 이 JD의 핵심 역량 세 가지를 추출하고, 내 경험 중 가장 잘 연결되는 스토리를 짚어줘."

**원칙 2 — AI 출력은 60점짜리 초안이다**  
받은 즉시 제출하지 않는다. 반드시 내 경험·언어·판단으로 덮어쓴다.

**원칙 3 — 잘 된 것은 그 자리에서 저장한다**  
좋은 프롬프트, 검증된 답변 구조, 유용한 분석 틀을 정리해 놓는다.

## 4. 취준 상황별 AI 활용 가이드

### 4-1. 이력서·자기소개서


```
역할: 너는 [타겟 직무] 채용 전문가야.
목적: 이 JD에서 가장 중요하게 보는 역량 3가지를 추출하고,
      내 경험 중 가장 잘 연결되는 스토리를 찾아줘.
내 경험: [프로젝트명, 역할, 결과 간단히]
JD: [붙여넣기]
```

---

### 4-2. 인터뷰 준비

**면접관 스타일별 전략 수립**

```
역할: 너는 [회사명] 인터뷰 코치야.
면접관 정보: [이름, LinkedIn 직함, 면접 스타일 힌트]
면접 유형: [기술 / 인성 / 케이스]
목적: 이 면접관 스타일에 맞는 전략과 예상 질문 5개, 답변 초안을 만들어줘.
내 주요 경험: [삼성화재 / 네이버 / AWS 인턴 중 관련된 것]
```

**면접 후 분석**

```
이건 내가 오늘 본 면접 녹음/메모야.
1. 잘 한 점과 아쉬운 점을 각각 3가지씩 분석해줘
2. 면접관이 기대했을 답변과 내가 실제 한 답변의 차이를 짚어줘
3. 다음 면접에서 반드시 보완해야 할 포인트는?
```

---

### 4-3. 기술 과제

**아키텍처 설계 요청**

```
역할: 너는 엔터프라이즈 [AWS/Azure] 솔루션 아키텍트야.
목적: [과제 요구사항]에 맞는 아키텍처를 설계해줘.
제약:
  - [규모, 예산, 운영 인력]
  - [RTO/RPO 요구사항]
  - [현재 기술 스택]
출력 형식:
  1) 아키텍처 개요
  2) 서비스 선택 근거
  3) 고려한 대안 및 탈락 이유
  4) 내가 놓쳤을 수 있는 포인트
```

**⚠️ 주의**  
AI가 제안한 서비스 선택은 반드시 공식 문서로 교차 검증한다.

---

### 4-4. 회사·JD 리서치

**포지션 비교**

```
[회사A]의 [직무명]과 [회사B]의 [직무명]을 비교해줘.
비교 기준: 고객 접점 유형, 기술 깊이 요구 수준, 커리어 성장 경로
내 강점: [핵심 경험 2~3줄]
어느 쪽이 나한테 더 맞는지 판단 근거와 함께 말해줘.
```

## 6. 프롬프트 라이브러리 (축적 중)

> 좋은 결과가 나올 때마다 여기에 추가한다.

### [자소서] JD 역량 추출 + 경험 연결

```
역할: 너는 [직무] 채용 전문가야.
이 JD에서 가장 중요하게 보는 역량 3가지를 추출하고,
내 경험 중 가장 잘 연결되는 스토리를 찾아줘.
내 경험: [간단히]
JD: [붙여넣기]
```

### [면접] 면접관 스타일별 전략

```
역할: 너는 [회사] 인터뷰 코치야.
면접관: [스타일 힌트]
면접 유형: [기술/인성/케이스]
내 주요 경험: [관련 경험]
→ 전략 + 예상 질문 5개 + 답변 초안
```

### [면접] 면접 후 분석

```
오늘 본 면접 메모야.
1. 잘한 점 / 아쉬운 점 각 3가지
2. 면접관 기대 vs 내 실제 답변 차이
3. 다음 면접 보완 포인트
```

### [기술 과제] 아키텍처 설계

```
역할: 엔터프라이즈 [클라우드] SA
목적: [과제 요구사항] 아키텍처 설계
제약: [규모/예산/RTO·RPO/기술스택]
출력: 개요 / 선택 근거 / 대안 및 탈락 이유 / 내가 놓쳤을 포인트
```

## 7. 앞으로 해볼 것

지금까지는 **반응형**으로 AI를 썼다 — 과제가 생기면 열고, 끝나면 닫는.  
앞으로는 **축적형**으로 전환한다.

**단기 (지금 당장)**

- 이 Playbook의 프롬프트 라이브러리를 Notion에 옮기기
- 다음 면접·과제마다 쓴 프롬프트를 바로 여기 추가하기

**중기 (취업 이후)**

- 취준에서 검증된 프롬프트 구조를 SA 직무로 확장
- 고객 제안서·기술 문서 작성에 같은 원칙 적용
