## 1. 선택한 키워드 (3개)
### 키워드 1: AI-native Development Platforms (AI 네이티브 개발 플랫폼)
 
| 항목 | 내용 |
|------|------|
| **간단 설명** | 생성형 AI가 코드 생성·테스트·배포까지 개발 생명주기 전반에 내재화된 플랫폼. "vibe coding"부터 엔터프라이즈급 AI 협업 개발까지 포괄 |
| **관련 AI 용어** | **LLM-assisted Coding**: Copilot 등 LLM이 코드를 자동 완성·생성하는 방식 / **Agentic Workflow**: 에이전트가 기획→코딩→테스트→배포를 자율 수행하는 흐름 / **Prompt Engineering**: 원하는 출력을 이끌어내기 위해 입력 프롬프트를 설계하는 기술 |

### 키워드 2: Multiagent Systems (멀티에이전트 시스템)
 
| 항목 | 내용 |
|------|------|
| **간단 설명** | 여러 AI 에이전트가 모듈 단위로 협력하거나 독립적으로 작동하며 복잡한 업무를 자동화하는 시스템 |
| **관련 AI 용어** | **Agent**: 목표 달성을 위해 자율적으로 행동하는 AI 모듈 / **Orchestration**: 여러 에이전트의 작업 순서와 흐름을 조율하는 메커니즘 / **Tool Use**: 에이전트가 외부 API·DB 등을 호출하는 기능 |

### 키워드 3: AIOps (AI 기반 운영 자동화)
 
| 항목 | 내용 |
|------|------|
| **간단 설명** | 시스템 운영(모니터링, 장애 대응, 로그 분석 등)을 AI로 자동화하는 기술 |
| **관련 AI 용어** | **Observability**: 시스템 상태를 추적하는 능력 / **Anomaly Detection:** 이상 탐지 / **Root Cause Analysis**: 장애 원인 분석 / **Auto-remediation**: 자동 복구 |

## 2. 왜 이 키워드를 선택했는가

### AI-native Development Platforms
 
- **선택 이유**: SA로서 고객에게 플랫폼 도입을 설계·권고하는 위치에 있음
- **직무 연결점**: 고객의 개발팀이 AI 네이티브 플랫폼으로 전환할 때, 기술 선택·보안·거버넌스를 함께 설계하는 것이 SA의 역할
- **직무 변화 예상**: 코딩을 직접 하지 않더라도 AI 개발 플랫폼의 아키텍처 패턴·제약 조건을 깊이 이해해야 설득력 있는 설계안을 낼 수 있음
 
### Multiagent Systems
 
- **선택 이유**: Microsoft Copilot Studio, Azure AI Foundry에서 직접 에이전트를 구축한 경험이 있어 가장 실질적으로 체감되는 트렌드
- **직무 연결점**: Solution Architect로서 클라이언트의 복잡한 비즈니스 프로세스를 에이전트 단위로 분해·설계하는 것이 핵심 역할이 됨
- **직무 변화 예상**: 단일 챗봇 설계에서 → 멀티에이전트 오케스트레이션 아키텍처 설계로 복잡도가 높아짐. 에이전트 간 신뢰·에러 처리·거버넌스 설계 역량이 필수화될 것

### AIOps
 
- **선택 이유**: 에이전트 시스템이 고객 환경에 배포되고 나면, 그 운영 단계에서 무슨 일이 일어나는지를 SA가 설계 시점부터 고려해야 한다는 문제의식에서 선택했다. 아키텍처 설계는 배포로 끝나지 않고 운영 가시성까지 포함해야 완결된다.
- **직무 연결점**: SA는 고객에게 솔루션을 제안할 때 "어떻게 만드는가"뿐 아니라 "배포 후 어떻게 관리되는가"까지 답해야 한다. AIOps는 그 운영 단계를 AI로 자동화하는 접근이므로, 설계 초기부터 Observability와 Auto-remediation 전략을 아키텍처에 반영해야 한다.
- **직무 변화 예상**: 로그·모니터링 설계가 단순 인프라 설정이 아닌 AI 추론의 입력 데이터로 취급되기 시작한다.

## 3. 나의 산업 / 직무 정의
**내가 속한 산업**: 엔터프라이즈 AI / 클라우드 기술 산업
**내 직무가 해결하는 문제**: 고객 조직이 AI 기술을 도입할 때 발생하는 기술 선택의 불확실성과 설계 복잡성을 줄여, 실제로 작동하는 AI 솔루션이 현업에 안착하도록 만든다.
**내가 만드는 결과물**: 고객의 비즈니스 요구사항을 AI 아키텍처로 변환한 설계 문서, PoC(개념 검증), 그리고 고객 스스로 운영·확장할 수 있는 기술 기반.
